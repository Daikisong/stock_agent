그럼 이걸 자동화하려면 어떻게 가능할거같아?

결국 코스피, 코스닥에 있는 기업에 대입해서 모든 종목에 대해서 지금같은 좋은 예시가 있으니깐 에이전트화 시킨다고하면 폭발적인 EPS 상승 후 리레이팅 < 이런 종목들을 찾는데는 확실히 도움이 될거니깐

가능합니다. 오히려 이건 **에이전트화하기 좋은 문제**예요.
왜냐하면 “폭발적인 EPS 상승 후 리레이팅”은 감이 아니라 **반복 가능한 패턴**이기 때문입니다.

핵심은 LLM에게 “좋은 종목 찾아줘”라고 바로 시키는 게 아닙니다.
먼저 전 종목을 숫자로 걸러내고, 그다음 에이전트가 공시·리포트·뉴스를 읽어서 **이 EPS 상승이 진짜 구조적 리레이팅인지, 그냥 일회성인지** 판별하게 만들어야 합니다.

한 문장으로 말하면:

> **퀀트 스크리너가 전 종목에서 후보를 뽑고,
> LLM 에이전트가 그 후보의 ‘리레이팅 스토리’를 검증하고,
> 밸류에이션 엔진이 아직 먹을 구간이 남았는지 점수화하는 구조**입니다.

---

# 1. 자동화의 목표 정의

찾으려는 종목은 단순 저PER주가 아닙니다.
또 단순 테마주도 아닙니다.

자동화 엔진이 찾아야 하는 건 이런 종목입니다.

> **현재 시장은 평범한 업종으로 보고 있는데,
> 향후 1~3년 EPS가 폭발적으로 커지고,
> 그 EPS가 일회성이 아니라 구조적 병목·수주·가격·믹스 변화에서 나오며,
> 아직 밸류에이션이 완전히 리레이팅되지 않은 종목.**

이걸 저는 시스템 안에서 이렇게 부를 수 있다고 봅니다.

> **E2R 후보: EPS Explosion → Valuation Re-rating**

# 2. 전체 구조

자동화하면 구조는 이렇게 갑니다.

| 단계   | 역할                                   |
| ---- | ------------------------------------ |
| 1단계  | 코스피·코스닥 전체 유니버스 수집                   |
| 2단계  | 가격, 시총, 거래대금, 재무제표, EPS, 컨센서스, 공시 수집 |
| 3단계  | 숫자 기반 1차 스코어링                        |
| 4단계  | 공시·뉴스·리포트 텍스트 분석                     |
| 5단계  | EPS 폭발 가능성 검증                        |
| 6단계  | 리레이팅 가능성 검증                          |
| 7단계  | 밸류에이션 매력도 평가                         |
| 8단계  | S/A/B/C 등급화                          |
| 9단계  | 매일/매주 변동 감지 알림                       |
| 10단계 | 과거 사례로 백테스트                          |

중요한 건 **숫자가 먼저, 에이전트 해석은 나중**입니다.
LLM이 먼저 종목을 고르게 하면 테마주에 끌립니다. 숫자로 먼저 압축하고, 그다음 LLM이 “왜 이 숫자가 나왔는지”를 읽어야 합니다.

# 3. 필요한 데이터 소스

## 1) 상장 종목 유니버스

코스피·코스닥 전체 종목, 시총, 거래량, 시장 구분, 업종은 KRX 데이터가 기본입니다. KRX 데이터 마켓은 상장종목 현황과 전종목 시세 정보를 제공합니다. ([KRX 데이터시스템][1])

## 2) 재무제표와 공시

DART가 핵심입니다. DART는 기업 공시를 전자적으로 제출·조회하는 시스템이고, OpenDART는 공시 원문, 주요 공시, 재무정보 등을 API로 활용할 수 있게 제공합니다. 특히 상장법인 KOSPI·KOSDAQ의 정기보고서 재무제표를 분기별로 받을 수 있습니다. ([DART][2])

## 3) 주가·시세

주가 데이터는 KRX, 증권사 API, 한국투자증권 Open API 같은 루트로 받을 수 있습니다. 한국투자증권 Open API는 국내주식 현재가, 일/주/월/년 기간별 시세, 재무비율, 손익계산서, 대차대조표 등 국내주식 관련 API 항목을 제공합니다. ([한국투자증권 API Portal][3])

## 4) 컨센서스 EPS

이게 제일 중요합니다.
“폭발적인 EPS 상승”을 제대로 잡으려면 과거 실적만 보면 안 됩니다. **2025E, 2026E, 2027E EPS 컨센서스와 그 상향 속도**를 봐야 합니다.

FnGuide DataGuide는 기업실적 컨센서스와 금융 데이터베이스를 제공하는 유료 데이터 서비스이고, WiseReport 쪽에서도 EPS 상향·하향 같은 컨센서스 변화를 확인할 수 있습니다. 다만 이런 데이터는 저작권·라이선스 이슈가 있으므로 자동 DB화는 정식 계약 기반으로 쓰는 게 안전합니다. ([DataGuide 고객지원][4])

## 5) 공시·투자유의·관리종목 필터

KIND는 한국거래소의 기업공시 채널이고, 상장법인 상세정보, 공시, 투자유의사항, 관리종목, 불성실공시법인, 투자주의·경고·위험종목 정보를 제공합니다. 전 종목 자동화에서는 이런 위험 필터가 필수입니다. ([KIND][5])

# 5. 1차 필터: 쓰레기 후보 제거

전 종목을 다 보면 노이즈가 너무 많습니다.

예를 들면:

| 필터                        | 이유                     |
| ------------------------- | ---------------------- |
| 우선주, 스팩, ETF, 리츠 제외       | 일반 기업 EPS 리레이팅 분석에 부적합 |
| 관리종목, 환기종목, 거래정지 제외       | 리스크 과다                 |
| 최근 20일 평균 거래대금 10억~30억 이상 | 매매 가능성 확보              |
| 시가총액 1,000억 이상            | 너무 작은 종목 노이즈 제거        |
| 감사의견 비적정, 자본잠식 위험 제외      | 숫자 좋아도 생존 리스크          |
| 부채비율 과도, 이자보상배율 낮은 종목 감점  | EPS가 나와도 재무 리스크 큼      |

단, “수십루타”를 노리려면 시총 필터를 너무 높게 잡으면 안 됩니다.
초기 알파는 보통 중소형에서 나옵니다. 그래서 저는 기본형은 **시총 1,000억 이상**, 공격형은 **시총 500억 이상**까지 열어두는 게 좋다고 봅니다.

# 6. 핵심 점수 1: EPS Explosion Score

이 엔진의 심장입니다.

단순히 “EPS 증가율 높음”이 아닙니다.
적자에서 흑자 전환하는 종목은 EPS 증가율 계산이 깨집니다. 그래서 여러 지표를 같이 봐야 합니다.

## 봐야 할 지표

| 지표                       | 의미                  |
| ------------------------ | ------------------- |
| 2026E EPS / 2024 EPS     | EPS가 몇 배 커지는지       |
| 2026E 영업이익 / 2024 영업이익   | 순이익 왜곡 제거           |
| 2026E 영업이익률 - 2024 영업이익률 | 마진 레벨업 여부           |
| 2026E ROE - 2024 ROE     | 자본 효율 개선            |
| 3개월 EPS 컨센서스 상향률         | 시장 추정치가 올라가는지       |
| 적자 → 흑자 전환 여부            | 턴어라운드 강도            |
| 영업이익 증가분 / 현재 시총         | 실적 증가가 시총 대비 얼마나 큰지 |

가장 중요한 건 마지막입니다.

예를 들어 어떤 회사의 영업이익이 300억에서 1,500억으로 늘어납니다.
증가분은 1,200억입니다.
그런데 시총이 2조면 임팩트가 약합니다.
시총이 4,000억이면 엄청난 임팩트입니다.

그래서 자동화에서는 이렇게 봐야 합니다.

> **EPS 증가율만 보지 말고, 영업이익 증가분이 현재 시총 대비 얼마나 큰지 봐야 한다.**

## 예시 공식

```text
EPS_Explosion_Score =
  0.25 × rank(2026E_OP / 2024_OP)
+ 0.20 × rank(2026E_EPS / 2024_EPS)
+ 0.20 × rank(OP_increase_2024_to_2026 / current_market_cap)
+ 0.15 × rank(OPM_expansion)
+ 0.10 × rank(ROE_expansion)
+ 0.10 × turnaround_bonus
```

적자 기업은 EPS 배율 대신 이렇게 처리합니다.

```text
if 2024_OP <= 0 and 2026E_OP > 0:
    use OP_delta / market_cap
    use 2026E_OPM
    add turnaround_bonus
```

이렇게 해야 하닉 2023년 같은 종목이 잡힙니다.
하닉은 적자라서 PER로는 안 보였지만, **흑자 전환 후 EPS 폭발**이 보이는 종목이었으니까요.

# 7. 핵심 점수 2: Revision Score

리레이팅 종목은 컨센서스가 위로 움직입니다.

주가가 오르기 전에 먼저 움직이는 게 있습니다.

> **애널리스트 EPS 추정치 상향**

진짜 좋은 종목은 이런 흐름이 나옵니다.

| 단계           | 의미       |
| ------------ | -------- |
| 처음에는 커버리지 없음 | 시장 관심 적음 |
| 일부 증권사 커버 시작 | 발굴 시작    |
| EPS 추정치 상향   | 숫자 반영 시작 |
| 목표주가 상향      | 기관 관심 증가 |
| 추정치 또 상향     | 본격 리레이팅  |

그래서 자동화에서는 매주 이런 걸 봐야 합니다.

```text
Revision_Score =
  0.35 × EPS_1M_revision
+ 0.25 × OP_1M_revision
+ 0.20 × EPS_3M_revision
+ 0.10 × analyst_count_increase
+ 0.10 × target_price_revision
```

특히 중요한 조건은:

```text
EPS 컨센서스는 올라가는데, 주가는 아직 크게 안 오른 종목
```

이게 제일 맛있는 구간입니다.
실적 추정치는 먼저 움직였는데 가격이 아직 못 따라온 상태니까요.

---

# 8. 핵심 점수 3: Valuation Sweet Spot

좋은 종목이어도 이미 너무 비싸면 늦습니다.

여기서는 PER만 보면 안 됩니다.
업종별로 적절한 지표가 다릅니다.

| 업종        | 주로 볼 지표                      |
| --------- | ---------------------------- |
| 메모리, 경기민감 | PBR, ROE, forward PER        |
| 조선, 방산    | 수주잔고, forward PER, EV/EBITDA |
| 화장품, 음식료  | PER, ROE, OPM, 해외 성장률        |
| 제약, CDMO  | PER, EBITDA, 수주계약, 가동률       |
| 금융        | PBR, ROE, 배당, 자사주            |
| 적자 턴어라운드  | OP 증가분 / 시총, EV/EBITDA       |

## 밸류에이션 점수 예시

```text
Valuation_Score =
  0.30 × rank(forward_earnings_yield)
+ 0.25 × rank(ROE_2026E / PBR_current)
+ 0.20 × rank(OP_increase / market_cap)
+ 0.15 × rank(PEG_inverse)
+ 0.10 × rank(peer_discount)
```

쉽게 말하면:

> **미래 ROE는 높은데 PBR은 아직 낮은 종목.
> 미래 EPS는 빠르게 커지는데 forward PER은 아직 낮은 종목.
> 영업이익 증가분이 현재 시총 대비 큰 종목.**

이게 자동화에서 제일 중요합니다.

---

# 9. 핵심 점수 4: Rerating Score

이건 숫자만으로 안 됩니다.
여기서 LLM 에이전트가 들어갑니다.

LLM이 공시, 사업보고서, IR, 리포트, 뉴스에서 이런 문장을 찾아야 합니다.

| 키워드                   | 의미        |
| --------------------- | --------- |
| 수주잔고 증가               | 향후 매출 가시성 |
| 가격 인상, ASP 상승         | 마진 개선     |
| 고부가 제품 비중 확대          | 믹스 개선     |
| 신규 고객 인증              | 매출처 확장    |
| 생산능력 증설               | 수요 확신     |
| 가동률 상승                | 영업 레버리지   |
| 공급 부족                 | 병목 프리미엄   |
| 장기 계약                 | 이익 지속성    |
| 해외 매출 증가              | 시장 확대     |
| 기존 업종에서 새 업종으로 재분류 가능 | 리레이팅 핵심   |

이걸 LLM이 JSON으로 뽑게 만들면 됩니다.

예시:

```json
{
  "rerating_thesis": "단순 반도체 장비주에서 AI 메모리 테스트 병목주로 재분류 가능",
  "evidence": [
    "고부가 제품 매출 증가",
    "주요 고객사 capex 확대",
    "2026E EPS 컨센서스 상향",
    "영업이익률 개선"
  ],
  "risk": [
    "고객사 집중도",
    "단기 주가 급등",
    "수주 공백 가능성"
  ],
  "stage": "discovery",
  "confidence": 0.74
}
```

이게 핵심입니다.
LLM은 종목을 찍는 도구가 아니라, **숫자로 걸러진 후보의 리레이팅 서사를 검증하는 도구**로 써야 합니다.

---

# 10. 핵심 점수 5: Price Stage Score

이것도 중요합니다.
좋은 종목이어도 이미 너무 많이 올랐으면 “졸업생”입니다.

그래서 가격 단계를 나눠야 합니다.

| 단계      | 상태                        | 의미      |
| ------- | ------------------------- | ------- |
| Stage 0 | EPS 추정치 상승, 주가 정체         | 최고 관심   |
| Stage 1 | 주가 20~80% 상승, EPS도 상향     | 발견 구간   |
| Stage 2 | 주가 100~200% 상승, 기관 커버 증가  | 리레이팅 진행 |
| Stage 3 | 주가 300% 이상 상승, PER/PBR 급등 | 졸업생 가능성 |
| Stage 4 | EPS 둔화, 가격만 상승            | 위험      |

이 구조로 보면 하닉은 초기에 Stage 0~1이었고, 지금은 Stage 2~3에 가깝습니다.
한미반도체, HD현대일렉트릭 같은 종목은 이미 Stage 3 성격이 강합니다.

## 점수화 예시

```text
if price_6m_return < 30% and EPS_revision_3m > 20%:
    Price_Stage_Score = 100

elif price_6m_return between 30% and 100% and EPS_revision_3m > 30%:
    Price_Stage_Score = 80

elif price_6m_return > 200% and valuation_expensive:
    Price_Stage_Score = 30

elif price_6m_return > 300% and EPS_revision_flat:
    Price_Stage_Score = 10
```

즉, **가격이 너무 안 오른 것도 문제지만, EPS보다 가격이 너무 먼저 간 것도 문제**입니다.

# 11. 최종 통합 점수

저라면 이렇게 가중치를 둡니다.

```text
Total_E2R_Score =
  0.30 × EPS_Explosion_Score
+ 0.20 × Valuation_Score
+ 0.15 × Revision_Score
+ 0.15 × Rerating_Score
+ 0.10 × Price_Stage_Score
+ 0.10 × Quality_Score
- Risk_Penalty
```

등급은 이렇게 줍니다.

| 등급      | 기준                                     |
| ------- | -------------------------------------- |
| **S**   | EPS 폭발, 밸류 매력, 리레이팅 서사, 컨센서스 상향이 모두 강함 |
| **A**   | 구조는 매우 좋지만 주가가 일부 반영됐거나 리스크 하나 있음      |
| **B**   | 방향은 맞지만 EPS·수주·밸류 중 하나가 아직 약함          |
| **C**   | 테마는 있지만 숫자가 약함                         |
| **졸업생** | 좋은 기업이지만 이미 리레이팅이 많이 진행됨               |
| **탈락**  | EPS 없음, 밸류 과열, 재무위험, 일회성 이익            |

# 12. “하닉 2023년 10월형”을 자동으로 찾는 법

우리가 진짜 찾고 싶은 건 이겁니다.

> **실적표는 아직 더럽지만, 다음 EPS가 폭발할 증거가 쌓이는 종목.**

자동화 조건으로 쓰면:

```text
조건 1: 최근 실적은 적자 또는 저마진
조건 2: 적자폭 축소 또는 영업이익률 개선 시작
조건 3: 고부가 제품, ASP 상승, 수주잔고, 가격 인상 키워드 등장
조건 4: 2025E/2026E EPS 컨센서스 상향 시작
조건 5: PBR 또는 forward PER이 아직 부담스럽지 않음
조건 6: 주가는 아직 6개월 100% 이상 급등하지 않음
```

이게 “하닉형”입니다.

반대로 “삼성전자 2024년 초형”은 이렇게 찾습니다.

```text
조건 1: 대형주 또는 업종 대표주
조건 2: 현재 실적은 바닥권
조건 3: PBR이 역사적 하단 또는 업종 평균 이하
조건 4: 업황 가격 반등 신호
조건 5: 시장은 아직 경쟁력 의심
조건 6: 특정 고부가 제품에서 후발 추격 신호
```

이건 “저평가 대형 턴어라운드형”입니다.

# 13. 에이전트 역할 분담

하나의 LLM에게 다 시키지 말고, 역할을 나누는 게 좋습니다.

## 1) Universe Agent

전 종목 목록, 시장 구분, 관리종목, 우선주, 스팩, 거래대금 필터를 관리합니다.

출력:

```json
{
  "eligible_universe": 1840,
  "excluded": {
    "spac": 85,
    "preferred": 120,
    "managed": 42,
    "illiquid": 390
  }
}
```

## 2) Quant Screener Agent

숫자만 봅니다.
감정 없이 EPS, OP, ROE, PER, PBR, 거래대금, 시총, 컨센서스 상향을 점수화합니다.

출력:

```json
{
  "code": "000000",
  "name": "예시기업",
  "eps_explosion_score": 91,
  "valuation_score": 78,
  "revision_score": 84,
  "total_quant_score": 86
}
```

## 3) DART Parser Agent

사업보고서, 분기보고서, 주요사항보고서에서 핵심 문장을 뽑습니다.

찾아야 할 항목:

```text
- 사업부별 매출
- 수주잔고
- 주요 고객
- 생산능력
- 가동률
- 제품가격
- 원재료 가격
- 신규 사업
- 증설
- 소송/우발채무
- 전환사채/신주인수권
```

## 4) Rerating Thesis Agent

이 종목이 “기존 업종”에서 “새 병목 업종”으로 재분류될 수 있는지 평가합니다.

예:

```text
기존: 일반 PCB 업체
재분류: AI 서버 SSD/메모리 모듈 PCB 병목주

기존: 화장품 유통사
재분류: 글로벌 K-뷰티 플랫폼

기존: 발전 기자재 업체
재분류: AI 전력 병목/발전 인프라 수혜주

기존: 방산 부품사
재분류: 글로벌 방산 수출 체인
```

## 5) Valuation Agent

세 가지 시나리오를 만듭니다.

```text
Bear: 기존 업종 PER 적용
Base: 현재 컨센서스 EPS × 업종 평균 PER
Bull: 리레이팅 PER 적용
```

예시:

```text
현재 주가: 20,000원
2026E EPS: 2,000원

Bear PER 8배 = 16,000원
Base PER 15배 = 30,000원
Bull PER 25배 = 50,000원
```

이렇게 하면 “먹을 구간이 남았는지”가 보입니다.

## 6) Red Team Agent

이 에이전트가 제일 중요합니다.
무조건 반대 논리를 찾게 해야 합니다.

찾아야 할 리스크:

```text
- 이미 주가가 너무 올랐는가?
- EPS 증가가 일회성인가?
- 원재료 가격 때문에 마진이 깨질 수 있는가?
- 수주가 특정 고객 하나에 집중됐는가?
- 전환사채, 유상증자 가능성이 있는가?
- 실제 현금흐름은 나쁜가?
- 재고가 쌓이고 있는가?
- 공시와 뉴스가 서로 모순되는가?
- 컨센서스 애널리스트 수가 너무 적은가?
```

이걸 안 넣으면 에이전트가 테마에 취합니다.

| ----------- | ------: |
| 2024 영업이익   |    150억 |
| 2026E 영업이익  |    900억 |
| 2024 EPS    |    500원 |
| 2026E EPS   |  3,000원 |
| 현재 주가       | 30,000원 |

| PBR         |    1.4배 |
| 2026E ROE   |     28% |
| 최근 6개월 주가   |    +22% |
| EPS 3개월 상향률 |    +35% |

| EPS 상향률       |   +5% |

# 19. 자동화에서 가장 조심해야 할 함정

## 1) 후행 PER 착시

하닉 2023년은 적자라 PER로 안 보였습니다.
삼전 2024년 초도 후행 실적은 예쁘지 않았습니다.

그래서 후행 PER만 보면 초대박 후보를 놓칩니다.

## 2) 적자 턴어라운드 계산 오류

적자에서 흑자 전환하면 EPS 증가율이 무한대처럼 보입니다.
그래서 반드시 `영업이익 증가분 / 시총`, `2026E 영업이익률`, `현금흐름`을 같이 봐야 합니다.

## 3) 컨센서스 없는 중소형주

코스닥 중소형주는 애널리스트 커버리지가 없을 수 있습니다.
그런 종목은 컨센서스가 없다고 버리면 안 됩니다.

대신 자체 추정 모델을 만들어야 합니다.

```text
수주잔고 × 매출전환율 × 예상 영업이익률 = 추정 영업이익
```

예를 들면 조선 기자재, 방산 부품, 플랜트 장비주는 이 방식이 더 잘 맞습니다.

## 4) 테마 키워드만 있는 종목

“AI”, “로봇”, “2차전지”, “원전”, “우주”라는 단어만으로는 부족합니다.
반드시 EPS로 내려와야 합니다.

## 5) 이미 오른 종목 추격

좋은 기업과 좋은 주식은 다릅니다.
자동화 엔진은 “좋은 기업”이 아니라 **아직 시장이 덜 가격 매긴 좋은 변화**를 찾아야 합니다.

# 21. 제일 중요한 설계 철학

이 자동화의 목적은 “정답 종목을 자동으로 매수”가 아닙니다.
목적은 이겁니다.

> **전 종목 중에서
> ‘하닉 2023년 10월형’,
> ‘삼전 2024년 초형’,
> ‘한미반도체 초입형’,
> ‘HD현대일렉트릭 초입형’,
> ‘삼양식품 초입형’
> 같은 패턴을 매주 자동으로 끌어올리는 것.**

이건 충분히 가능합니다.

다만 성공하려면 세 가지가 필수입니다.
