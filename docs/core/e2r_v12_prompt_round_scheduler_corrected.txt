이 프롬프트는 **가격경로 돌파 이후** 사용하는 historical calibration / backtest optimization 전용 프롬프트다.

v11의 핵심은 유지한다.

```text

source_basis = FinanceData/marcap transformed into assistant-readable symbol-year CSV shards
```

이 프롬프트는 현재/live 종목 탐색용이 아니다.
이 프롬프트는 `stock_agent` 레포 패치용도 아니다.

# 단일 프롬프트 시작

이번 작업은 코딩 작업도 아니고, 레포 반영 작업도 아니고, 현재 live 후보 탐색도 아니다.

```text
universe = atlas/universe/all_symbols.csv
```

`manifest`에서 반드시 확인한다.

```text

price_adjustment_status

tradable_row_count
raw_row_count
symbol_count
active_like_symbol_count
inactive_or_delisted_like_symbol_count

calibration_shard_root

schema_path

```

중요:

```text
- max_date는 매번 manifest에서 읽는다.
- 현재 날짜로 max_date를 추정하지 않는다.
- trigger forward window는 current_date가 아니라 stock-web manifest max_date 기준으로 판단한다.
- manifest max_date 이후 가격은 만들지 않는다.
```

ticker는 6자리 문자열로 정규화한다.

```text
ticker = six_digit_code
```

```text

l  = low

mc = market_cap

m  = market
```

기본 price basis:

```text

calibration_shard_root = atlas/ohlcv_tradable_by_symbol_year
```

주의:

```text
- corporate action contaminated window는 기본적으로 calibration_usable=false 처리한다.
```


```text
if entry_date not in tradable shard:
    use next tradable date as entry_date only if evidence timing rule allows next-trading-day entry

    calibration_usable = false
    reason = insufficient_forward_window_in_stock_web

if corporate_action_candidate_dates overlap entry_date~D+180 window:
    calibration_usable = false
    reason = corporate_action_contaminated_180D_window

if corporate_action_candidate_dates overlap D+252 or D+504 but not D+180:
    30D/90D/180D usable may remain true
    1Y/2Y fields must be marked contaminated_or_unavailable
```

raw shard 사용 원칙:

```text
- 기본은 tradable shard만 사용한다.
- raw shard는 why tradable row is missing, row_status 확인, zero-volume/zero-OHLC/invalid row 진단, narrative-only 또는 blocked reason 설명에만 쓴다.
- raw shard로 weight calibration을 하지 않는다.
```

## 4. Coverage-Index Scheduler / Large Sector ID

v12 연구는 R1~R13을 무조건 차례대로 돌리지 않는다. 먼저 `docs/core/V12_Research_No_Repeat_Index.md`를 읽고, 아키타입별 row 수량과 중복 상태를 기준으로 연구 대상을 고른다. Round는 선택된 canonical_archetype_id가 속한 sector를 파일명과 metadata에 일관되게 적기 위한 분류값이다.

핵심 원칙:

```text
round_scheduler_priority = coverage_index_first
sequential_round_cycle_required = false
coverage_gap_can_override_previous_round = true
selected_archetype_drives_round = true
manual_round_assignment_required = false
```

쉬운 예:

```text
직전 연구가 R8이었다고 다음을 반드시 R9로 가지 않는다.
V12_Research_No_Repeat_Index.md에서 C08이 14 row이고 C29가 86 row라면,
다음 연구는 C08 같은 부족 아키타입을 먼저 선택한다.
선택 결과 C08이면 output filename의 round는 R2가 된다.
```

### 4.1 Coverage index 필수 확인

연구 시작 전 반드시 아래 문서를 먼저 읽는다.

```text
required_first_read:
- docs/core/V12_Research_No_Repeat_Index.md
```

이 문서에서 확인할 항목:

```text
1. Priority 0: 30 row 미만 canonical_archetype_id
2. Priority 1: 30~49 row canonical_archetype_id
3. Priority 2: 이미 50 row 이상인 과다/충분 coverage 구역
4. 반복 위험이 높은 symbol/archetype 조합
5. exact duplicate key 예시
6. evidence_url_pending / source_proxy_only 보강 필요 여부
```

### 4.2 Archetype-first selection

연구 대상 선택 절차:

```text
1. V12_Research_No_Repeat_Index.md의 Priority 0 표를 본다.
2. Priority 0이 남아 있으면 rows가 가장 낮은 canonical_archetype_id를 우선 선택한다.
3. 같은 row 수라면 positive/counterexample/4B/4C 균형이 더 나쁜 쪽을 먼저 선택한다.
4. Priority 0이 모두 30 row 이상이면 Priority 1에서 50 row까지 끌어올릴 대상을 고른다.
5. Priority 2는 새 failure mode, 새 market regime, URL 검증 보강, hard 4C 확인이 있을 때만 선택한다.
6. 선택한 canonical_archetype_id에서 새 symbol, 새 trigger family, counterexample, 4B/4C path를 찾는다.
7. hard duplicate인 canonical_archetype_id + symbol + trigger_type + entry_date 조합은 쓰지 않는다.
```

우선순위 점수:

```text
priority_0_under_30_rows_bonus = +100
priority_1_under_50_rows_bonus = +60
new_symbol_bonus = +20
new_trigger_family_bonus = +15
counterexample_gap_bonus = +15
positive_gap_bonus = +12
missing_4b_or_4c_path_bonus = +10
verified_url_repair_bonus = +8
source_proxy_replacement_bonus = +8
priority_2_over_50_rows_penalty = -20
r13_general_repeat_penalty = -50
hard_duplicate_penalty = -999
```

### 4.4 Loop number rule

loop 번호는 순환 스케줄 완성도를 맞추기 위한 값이 아니라 output 식별자다.

```text
selected_loop = max(existing loop for selected_round and selected_canonical_archetype_id) + 1
if no prior file exists for that pair:

if no prior v12 file exists:
```


### 4.7 Output filename hard rule

파일명은 반드시 selected_round와 selected_loop를 따른다.


파일명은 위 패턴과 정확히 맞아야 한다.

```text

compact_filename_forbidden = true
```

금지 파일명 예시:

```text
c01_r1_loop97_shipbuilding_backlog_margin_bridge_1be59adb.md
c07_r2_loop96_hbm_equipment_order_relative_strength_974b1d7b.md
round_R1_loop_100_C01_research.md
```

위처럼 짧은 파일명이나 해시형 파일명은 사람이 보기에는 연구자료처럼 보여도 batch ingest에서 V12 result로 색인되지 않을 수 있다.

쉬운 예:

```text
좋음: e2r_stock_web_v12_residual_round_R1_loop_116_L1_INDUSTRIALS_INFRA_DEFENSE_GRID_C01_ORDER_BACKLOG_MARGIN_BRIDGE_research.md
나쁨: c01_r1_loop116_backlog.md
```

파일명, metadata, final response의 round/loop가 서로 다르면 실패다.

```text
if filename_round != metadata_round:
    output_invalid = true

if filename_loop != metadata_loop:
    output_invalid = true

if metadata_round != selected_round:
    output_invalid = true

if metadata_loop != selected_loop:
    output_invalid = true

if filename does not match required_filename_regex:
    output_invalid = true
    do_not_write_md = true
```

## 6.5 Coverage-Index Diversity Governor / Anti-Repetition Gate

이번 v12 연구의 목적은 기존 global calibration을 다시 증명하는 것이 아니라, `V12_Research_No_Repeat_Index.md`에서 부족한 canonical_archetype_id를 골라 새로운 symbol / trigger family / 실패 유형 / 4B·4C path / residual error를 채워 넣는 것이다.

핵심 원칙:

```text
round_scheduler_priority = coverage_index_first
coverage_gap_priority = global_archetype_gap_first
sequential_round_cycle_required = false
same_canonical_archetype_research = allowed_and_required
same_symbol_same_trigger_date_research = low_value_or_duplicate
same_symbol_new_trigger_family = allowed_with_reuse_reason
same_archetype_new_symbol_or_counterexample = high_value
```

중요:

```text
- coverage gap이 크면 직전 round와 달라도 부족 아키타입을 먼저 선택한다.
- R13은 순서가 와서 하는 round가 아니라 cross-archetype checkpoint가 필요할 때만 선택한다.
- R13은 개별 sector 연구가 아니라 cross-archetype checkpoint다.
- canonical_archetype_id는 연구의 기본 단위다. 같은 canonical_archetype을 여러 번 연구하는 것은 정상이다.
- 문제는 같은 canonical_archetype 반복이 아니라, 같은 symbol + 같은 trigger_date + 같은 entry_date + 같은 evidence family를 무한히 재검증하는 것이다.
```

### 6.5.1 Index-first selection

사용자가 명시적으로 round/loop를 주지 않으면 `## 4. Coverage-Index Scheduler`를 먼저 실행한다.

```text
selection_mode = coverage_index_then_round_metadata
manual_round_assignment_required = false
manual_parallel_session_assignment_required = false
auto_coverage_gap_can_override_round = true
```

선택 순서:

```text
1. docs/core/V12_Research_No_Repeat_Index.md를 읽는다.
2. Priority 0 / Priority 1 / Priority 2 표를 만든다.
3. 가장 부족한 canonical_archetype_id를 먼저 고른다.
4. 선택한 canonical_archetype_id에서 selected_round와 large_sector_id를 파생한다.
5. 그 canonical_archetype_id 안에서 새 symbol / trigger family / counterexample / 4B·4C path를 선택한다.
```

금지:

```text
- R1~R13 순환 순서만 보고 연구 대상을 고르지 마라.
- Priority 0/1이 남아 있는데 Priority 2의 과다 coverage 아키타입을 일반론으로 반복하지 마라.
- R13 파일명으로 일반 sector-specific MD를 만들지 마라.
- C20을 선택했다면 R5/L5로 쓰고, C22를 선택했다면 R6/L6로 쓴다.
```

### 6.5.2 Coverage 선택

`V12_Research_No_Repeat_Index.md`를 기준으로 canonical_archetype_id별 coverage table을 만든다.

우선순위:

```text
priority_1 = 30 row 미만 canonical_archetype_id
priority_2 = 50 row 미만 canonical_archetype_id
priority_3 = counterexample_count 또는 positive_case_count가 부족한 canonical_archetype
priority_4 = 4B 또는 4C case가 부족한 canonical_archetype
priority_5 = current calibrated profile이 틀린 residual_error가 있는 구역
priority_6 = 기존에 등장하지 않은 symbol 또는 trigger family
priority_7 = fine_archetype을 canonical_archetype으로 압축할 수 있는 구역
```

### 6.5.3 다양성 점수 — archetype 반복은 허용, symbol/trigger 반복은 차단

각 후보 case / trigger에 대해 diversity_score를 계산하고, 점수가 높은 후보를 우선한다.

```text
diversity_score =
  same_archetype_new_symbol_bonus
  + same_archetype_counterexample_bonus
  + same_archetype_new_trigger_family_bonus
  + same_archetype_new_regime_bonus
  + new_symbol_bonus
  + counterexample_gap_bonus
  + residual_error_bonus
  - wrong_selected_scope_penalty
  - repeated_same_symbol_penalty
  - repeated_same_trigger_date_penalty
  - repeated_same_entry_group_penalty
  - repeated_same_evidence_family_penalty
  - schema_rematerialization_penalty
```

점수 규칙:

```text
same_archetype_new_symbol_bonus = +4
same_archetype_counterexample_bonus = +5
same_archetype_new_trigger_family_bonus = +4
same_archetype_new_regime_bonus = +3
new_symbol_bonus = +3
counterexample_gap_bonus = +4
residual_error_bonus = +5

wrong_selected_scope_penalty = -999
repeated_same_symbol_penalty = -5
repeated_same_trigger_date_penalty = -5
repeated_same_entry_group_penalty = -6
repeated_same_evidence_family_penalty = -3
schema_rematerialization_penalty = -8
```

중요:

```text
- same canonical_archetype_id 자체에는 penalty를 주지 않는다.
- canonical_archetype_id가 같아도 symbol, trigger family, failure mode, market regime이 다르면 좋은 연구다.
- canonical_archetype_id가 같고 symbol과 trigger_date까지 같으면 낮은 가치 반복이다.
- 선택한 canonical_archetype_id와 맞지 않는 large_sector_id/round를 쓰면 wrong_selected_scope_penalty를 받아 선택될 수 없다.
```

### 6.5.4 반복 연구 금지

다음은 새 연구로 인정하지 않는다.

```text
- 같은 symbol + 같은 canonical_archetype_id + 같은 trigger_date + 같은 entry_date 재검증
- 같은 symbol + 같은 evidence family + 같은 entry_price 재검증
- 같은 R1/R2 HBM/방산/전력기기 대표 set을 같은 trigger family로 반복
- 이미 적용된 global rule을 같은 symbol과 같은 trigger로 다시 증명
- Stage2가 Green보다 빠르다는 일반론 반복
- price-only blowoff는 차단해야 한다는 일반론 반복
- 4B는 non-price evidence가 필요하다는 일반론 반복
- previous loop의 trigger table을 v12 field에 맞춰 다시 번호만 바꾸는 작업
- 선택한 canonical_archetype_id와 맞지 않는 round/large_sector로 sector-specific 연구를 작성하는 작업
```

위에 해당하면:

```text
loop_contribution_label = duplicate_low_value_loop
do_not_propose_new_weight_delta = true
independent_evidence_weight = 0.0
do_not_count_as_new_case = true
```

### 6.5.5 최소 신규성 조건 — 같은 archetype 안의 새 표본을 인정

각 MD는 아래 조건을 만족해야 한다.

```text
minimum_new_independent_case_ratio = 0.60
minimum_new_symbol_count = 2
minimum_counterexample_count = 1
minimum_positive_case_count = 1
```

calibration-usable case가 5개라면 최소 3개는 새 독립 case여야 한다.

새 독립 case는 아래 중 하나 이상을 만족해야 한다.

```text
- 같은 canonical_archetype_id 안에서 이전 연구에 없던 symbol
- 같은 canonical_archetype_id 안에서 이전 연구에 없던 trigger family
- 같은 canonical_archetype_id 안에서 이전 연구에 없던 실패 유형
- 같은 canonical_archetype_id 안에서 이전 연구에 없던 4B/4C path
- 같은 canonical_archetype_id 안에서 다른 market regime 또는 cycle phase
- 기존 global rule의 새로운 counterexample
- current calibrated profile이 틀린 residual error
- 이전 loop에 없던 canonical_archetype_id
- 이전 loop에 없던 sector propagation path
```

### 6.5.6 재사용 허용 조건

기존 case를 재사용할 수 있는 경우는 아래로 제한한다.

```text
- 같은 symbol이지만 전혀 다른 trigger family를 검증하는 경우
- 같은 symbol이지만 Stage2 entry가 아니라 4B/4C timing을 새롭게 검증하는 경우
- 기존 applied global rule의 반례로 쓰는 경우
- corporate-action / forward-window / price-row 오류를 고치는 경우
- holdout validation으로 명시한 경우
```

재사용 case는 반드시 아래 필드를 채운다.

```text
reuse_reason:
independent_evidence_weight:
do_not_count_as_new_case:
```

기본값:

```text
if reused_symbol and reused_trigger_date and reused_entry_date:
    independent_evidence_weight = 0.0
    do_not_count_as_new_case = true
    loop_contribution_label = schema_rematerialization_only

if reused_symbol but new_trigger_family:
    independent_evidence_weight = 0.25_to_0.5
    do_not_count_as_new_case = false

if same_canonical_archetype but new_symbol:
    independent_evidence_weight = 1.0
    do_not_count_as_new_case = false
```

아래 조건이면 이번 MD는 연구 산출물로는 남기되, 새 calibration weight 후보로 쓰지 않는다.

```text

    loop_contribution_label = scope_consistency_violation
    do_not_propose_new_weight_delta = true

if invalid_round_sector_pair == true:
    loop_contribution_label = scope_consistency_violation
    do_not_propose_new_weight_delta = true

if new_independent_case_count == 0:
    loop_contribution_label = duplicate_low_value_loop
    do_not_propose_new_weight_delta = true

if counterexample_count == 0 and positive_case_count > 0:
    counterexample_search_incomplete = true
    do_not_propose_global_delta = true

if calibration_usable_case_count < 3:
    do_not_propose_new_weight_delta = true

if all_cases_are_reused_without_new_trigger_family:
    loop_contribution_label = schema_rematerialization_only
    do_not_propose_new_weight_delta = true
```

이번 MD의 기여도를 한 줄로 요약한다.


## 7. New Loop Novelty Requirement

각 새 loop는 selected canonical_archetype_id 안에서 아래 조건을 만족해야 한다.

```text

coverage_index_selection_must_be_documented = true
round_sector_consistency_must_pass = true
```

calibration-usable case 중 최소 60%는 새 독립 case여야 한다.

새 독립 case는 아래 중 하나 이상이어야 한다.

```text
- selected canonical_archetype_id의 이전 연구에 없던 symbol
- selected canonical_archetype_id의 이전 연구에 없던 trigger family
- selected canonical_archetype_id의 이전 연구에 없던 실패 유형
- selected canonical_archetype_id의 이전 연구에 없던 4B/4C path
- selected canonical_archetype_id의 다른 market regime 또는 cycle phase
- 기존 applied scoring axis의 새로운 counterexample
- current calibrated profile이 틀린 residual error
- V12 index Priority 0/1에서 이전 loop에 없던 canonical_archetype_id
- selected large_sector_id 안에서 이전 loop에 없던 sector propagation path
```

중요:

```text
같은 canonical_archetype_id를 다시 쓰는 것은 정상이다.
같은 canonical_archetype 안에서 symbol / trigger family / counterexample / 4B / 4C coverage를 늘리는 것이 v12의 목적이다.
문제는 같은 symbol + 같은 trigger_date + 같은 entry_date + 같은 evidence를 반복하는 것이다.
하지만 어떤 경우에도 selected canonical_archetype_id와 맞지 않는 round/large_sector_id를 쓰면 안 된다.
```

같은 case를 재사용할 수 있는 경우:

```text
- 기존 row의 계산 오류를 수정할 때
- corporate-action / forward-window / dedupe 오류를 고칠 때
- 같은 symbol의 완전히 다른 trigger family를 새로 검증할 때
- 같은 symbol의 Stage2 entry가 아니라 4B/4C timing을 새로 검증할 때
- 기존 applied scoring axis의 counterexample으로 명시할 때
- holdout validation에서 대표 case로 재사용할 때
```

같은 case를 재사용하면 반드시 기록한다.

```text
reuse_reason
independent_evidence_weight = 0 | 0.25 | 0.5 | 1.0
do_not_count_as_new_case = true/false
```

if new_independent_case_ratio < 0.60:

```text

do_not_propose_new_weight_delta = true
```

if round_schedule_status != coverage_index_selected or round_sector_consistency != pass:

```text
loop_contribution_label = scope_consistency_violation
do_not_propose_new_weight_delta = true
```

## 11. Historical Eligibility Gate

각 calibration trigger는 아래 조건을 만족해야 한다.

```text

- entry_date 이후 최소 180 trading days가 stock-web에 존재한다.
```

## 13. MFE / MAE 계산식

```text
MFE_N_pct = (max(high from entry_date through N trading days) / entry_price - 1) * 100
MAE_N_pct = (min(low from entry_date through N trading days) / entry_price - 1) * 100
below_entry_price_flag_N = any(close < entry_price after entry_date within N trading days)
peak_price = max(high over observed window after entry_date)
peak_date = date of peak_price
drawdown_after_peak_pct = (min(low after peak_date) / peak_price - 1) * 100
```

N 기준:

```text
30D  = 30 trading days
90D  = 90 trading days
180D = 180 trading days
1Y   = 252 trading days
2Y   = 504 trading days
```

## 16. Stage3 Yellow / Green Lateness Audit

Stage3-Yellow와 Stage3-Green trigger의 MFE/MAE를 Stage2 / Stage2-Actionable과 비교한다.

반드시 계산한다.

```text
green_lateness_ratio =
(Stage3_Green_entry_price - Stage2_Actionable_entry_price)
/
(peak_price_after_Stage2_Actionable - Stage2_Actionable_entry_price)
```

해석:

```text
0.0~0.3 = Green이 크게 늦지 않음
0.3~0.6 = Green이 다소 늦음
0.6~1.0 = Green이 upside 대부분을 놓침
1.0 이상 = Green이 peak 이후 또는 거의 고점에서 뜸
```

Stage3-Green trigger가 없으면:

```text
green_lateness_ratio = not_applicable
reason = no_confirmed_Stage3_Green_trigger
```

## 17. 4B Timing Audit

4B는 “매도 신호” 하나가 아니라 Stage3 thesis 위에 붙는 overheat / dilution / valuation / legal / positioning / execution-risk overlay다.

가격만으로 local peak가 보였다고 full 4B로 확정하지 않는다.

기존 단일 `four_b_peak_proximity`는 사용하지 않는다. 반드시 local peak와 full observed-cycle peak를 분리한다.

```text
four_b_local_peak_proximity =
(Stage4B_entry_price - Stage2_Actionable_entry_price)
/
(local_peak_price_after_Stage2_Actionable - Stage2_Actionable_entry_price)
```

```text
four_b_full_window_peak_proximity =
(Stage4B_entry_price - Stage2_Actionable_entry_price)
/
(full_window_peak_price_after_Stage2_Actionable - Stage2_Actionable_entry_price)
```

판정 규칙:

```text
if four_b_local_peak_proximity is high and four_b_full_window_peak_proximity is low:
    four_b_timing_verdict = price_only_local_4B_too_early

if price-only local peak has no non-price evidence:
    do_not_treat_as_full_4B = true

if non-price 4B evidence exists and full-window proximity is 0.7~1.1:
    four_b_timing_verdict = good_full_window_4B_timing
```

`four_b_evidence_type`은 반드시 아래 중 하나 이상으로 기록한다.

```text
price_only
valuation_blowoff
revision_slowdown
dilution_or_cb
capital_raise_or_overhang
contract_delay
legal_or_regulatory_block
margin_or_backlog_slowdown
positioning_overheat
control_premium_or_event_premium
```

## 20. Sector-specific rule proposal

이번 v12 연구에서 제안하는 rule은 기본적으로 global이 아니라 sector/canonical archetype scope다.

```text
rule_scope = sector_specific | canonical_archetype_specific | global_candidate | no_new_rule
```

global_candidate는 매우 엄격하다.

```text
global_candidate 조건:
- 3개 이상 large_sector에서 반복
- unique_case_count >= 8
- counterexample 통과
- 기존 applied global axis와 중복 아님
```

sector_specific 조건:

```text
- 같은 large_sector 안에서 unique_case_count >= 3
- positive와 counterexample이 둘 다 있음
- MFE/MAE 방향성이 설명 가능함
- 기존 global rule보다 더 세밀한 설명을 제공함
```

canonical_archetype_specific 조건:

```text
- 같은 canonical_archetype_id 안에서 positive >= 2 또는 positive+counterexample >= 3
- evidence field 차이가 가격경로 차이를 설명함
```
