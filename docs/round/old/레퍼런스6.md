# 1. 전체 구조

에이전트는 한 개가 아니라 여러 개의 공정으로 나눠야 해.

```text
[0] 데이터 수집
    ↓
[1] 유니버스/리스크 필터
    ↓
[2] 가격·수급·상대강도 분석
    ↓
[3] 재무·컨센서스 분석
    ↓
[4] 공시·사업보고서·수주·CAPA 분석
    ↓
[5] 뉴스·산업 레짐 분석
    ↓
[6] 리레이팅 서사 분석
    ↓
[7] 레드팀 리스크 분석
    ↓
[8] 1차·2차·3차·4차 단계 분류
    ↓
[9] 매일 아침 브리핑 생성
```

중요한 순서는 **숫자 → 공시 → 뉴스 → 해석**이야.
LLM에게 처음부터 “좋은 종목 골라줘”라고 하면 테마주에 끌려가. 먼저 전 종목을 숫자로 압축하고, 그다음 공시와 리포트를 읽혀야 해.

# 2. 데이터 소스: 정확히 어디서 뭘 가져올지

## A. 종목 유니버스와 주가

| 데이터                    | 1차 출처                               | 용도                       |
| ---------------------- | ----------------------------------- | ------------------------ |
| 코스피·코스닥 전체 종목          | KRX Data Marketplace / KRX Open API | 분석 대상 유니버스 구성            |
| 일별 OHLCV, 거래대금, 시총     | KRX Open API, 한국투자증권 Open API       | 상승률, 상대강도, 유동성, 가격 단계 판정 |
| 현재가·실시간 가격             | 한국투자증권 Open API / 웹소켓               | 장중 알림, 전일 대비 변동          |
| PER/PBR/배당수익률          | KRX, FnGuide, QuantiWise            | 밸류에이션 필터                 |
| 투자주의·경고·위험, 관리종목, 거래정지 | KIND, KRX                           | 리스크 필터                   |

KRX Open API는 유가증권 일별매매정보 같은 주식 매매 데이터를 제공하고, KRX 쪽 설명상 2010년 이후 데이터를 제공하는 항목들이 있다. 한국투자증권 Open API도 국내주식 현재가와 기간별 시세 API를 제공한다. ([Open API][1])

KIND는 공시뿐 아니라 관리종목, 투자주의환기종목, 매매거래정지종목, 불성실공시법인, 투자주의·경고·위험종목 같은 투자유의사항 메뉴를 제공한다. 이건 E2R 후보에서 “좋아 보여도 제외하거나 감점해야 하는 종목”을 걸러내는 안전벨트다. ([KIND][2])

---

## B. 공시와 재무제표

| 데이터              | 출처                           | 용도                             |
| ---------------- | ---------------------------- | ------------------------------ |
| 기업 고유번호, 종목코드 매핑 | OpenDART `corpCode`          | DART 공시와 KRX 종목코드 연결           |
| 공시 검색            | OpenDART 공시검색 API            | 신규 수주, 시설투자, 증권발행, 실적공시 탐지     |
| 정기보고서 원문         | OpenDART / DART              | 사업부, 수주잔고, CAPA, 제품 가격, 리스크 추출 |
| 단일회사 전체 재무제표     | OpenDART `fnlttSinglAcntAll` | 매출, 영업이익, 순이익, 자산, 부채 등        |
| 주요 재무지표          | OpenDART 주요 재무지표 API         | ROE, 부채비율, 이익률 등               |

OpenDART는 DART 공시 원문을 XML 형식으로 활용할 수 있게 하고, 주요공시와 재무정보도 API로 제공한다. 고유번호 API는 `corp_code`, `corp_name`, `stock_code` 같은 매핑 정보를 제공한다. 공시검색 API는 시작일, 종료일, 공시유형, 법인 고유번호 등을 인자로 받아 공시 목록을 조회할 수 있다. ([OPENDART][3])

정기보고서 재무정보 쪽에서는 단일회사 전체 재무제표 API가 있고, 사업연도와 보고서 코드가 필요하다. 보고서 코드는 1분기 `11013`, 반기 `11012`, 3분기 `11014`, 사업보고서 `11011` 구조로 사용한다. ([OPENDART][4])

OpenDART는 일반적으로 20,000건 이상 요청 시 제한 초과가 날 수 있다는 안내도 있으니, 전 종목 매일 스캔에는 캐시와 증분 수집이 필수다. ([OPENDART][5])

## C. 컨센서스와 리포트

여기가 제일 중요해. E2R은 현재 실적보다 **미래 EPS의 상향**을 잡는 시스템이기 때문이야.

| 데이터                         | 권장 출처                          | 용도             |
| --------------------------- | ------------------------------ | -------------- |
| 2025E/2026E/2027E 매출·OP·EPS | FnGuide DataGuide / QuantiWise | EPS 폭발 점수      |
| 컨센서스 상향·하향                  | WiseReport, QuantiWise         | Revision Score |
| 애널리스트 수 변화                  | FnGuide / QuantiWise           | 커버리지 확산        |
| 목표주가 변화                     | FnGuide / WiseReport           | 시장 인식 변화       |
| 리포트 원문                      | WiseReport, 증권사 공식 홈페이지, IRGO  | 리레이팅 서사 추출     |

FnGuide DataGuide는 재무, 주가, 산업별 시계열 데이터 등을 제공하는 금융 데이터 분석 솔루션이고, QuantiWise는 컨센서스 데이터를 제공한다. QuantiWise 설명에는 일간 컨센서스 데이터가 2012년 1월 2일부터 제공되고, 90일 이내 발표 리포트를 취합하며, 컨센서스 최종 업데이트가 익일 오전 7시 30분경이라고 되어 있다. ([DataGuide][6])

WiseReport에는 주간 컨센서스 동향, EPS 상향·하향 같은 페이지가 있다. 다만 FnGuide/WiseReport 데이터는 저작권과 데이터베이스화 제한이 있으므로, 자동화 시스템에 넣으려면 정식 라이선스가 필요하다. ([WiseReport][7])

## D. 뉴스와 산업 레짐

| 데이터       | 출처                                      | 용도         |
| --------- | --------------------------------------- | ---------- |
| 글로벌 산업 뉴스 | Reuters API, Bloomberg, WSJ, FT, CNBC 등 | 섹터 레짐 변화   |
| 국내 뉴스     | 연합뉴스, 한국경제, 매일경제, 이데일리, 전자신문, 더벨 등      | 국내 기업 이벤트  |
| 뉴스 검색 API | 네이버 뉴스 검색 API, NewsAPI 등                | 키워드 탐색     |
| 공식 기관 자료  | IEA, EIA, DOE, 산업부, KRX, 기업 IR          | 테마의 실제성 검증 |

네이버 뉴스 검색 API는 애플리케이션 등록 후 클라이언트 ID와 시크릿을 발급받아 뉴스 검색을 실행하는 방식이다. Reuters도 별도의 API 통합 서비스를 제공한다. ([네이버 개발자][8])

산업 레짐 판단에는 “뉴스 한 개”가 아니라 **공식 통계 + 글로벌 뉴스 + 국내 리포트**가 같이 들어가야 한다. 예를 들어 변압기 사이클에서는 IEA가 데이터센터 전력 소비가 2030년 약 945TWh까지 두 배 이상 증가할 수 있다고 제시했고, Reuters 계열 보도는 미국 전력 인프라 확장이 변압기 부족으로 지연되고 있다는 점을 부각했다. ([IEA][9])

# 6. 에이전트별 역할

## 6-1. Universe Agent

역할은 “분석 가능한 종목만 남기기”야.

제외 조건:

```text
- 우선주
- 스팩
- 리츠
- ETF/ETN
- 거래정지
- 관리종목
- 투자주의환기종목
- 상장폐지 실질심사
- 최근 20일 평균 거래대금 너무 낮은 종목
- 감사의견 비적정
- 자본잠식 위험
```

공격형으로는 시총 500억 이상, 기본형으로는 시총 1,000억 이상, 기관형으로는 시총 3,000억 이상부터 보면 돼.

---

## 6-2. Price Agent

주가가 얼마나 움직였는지, 이미 졸업생인지 판정한다.

계산값:

```text
return_1d
return_5d
return_20d
return_60d
return_120d
return_250d
relative_return_vs_kospi
relative_return_vs_sector
distance_from_52w_high
distance_from_52w_low
trading_value_20d_avg
volume_spike
market_cap_change
```

가격 단계는 이렇게 본다.

| 가격 상태                     | 의미          |
| ------------------------- | ----------- |
| 6개월 +0~30%, EPS 상향        | 가장 좋은 잠복 구간 |
| 6개월 +30~100%, EPS 상향 동행   | 발견 구간       |
| 6개월 +100~250%, EPS 계속 상향  | 리레이팅 진행 구간  |
| 12개월 +300% 이상, PER/PBR 급등 | 졸업생 후보      |
| 가격은 급등, EPS 상향 없음         | 테마 과열       |

중요한 건 **많이 올랐다고 자동 탈락시키면 안 된다**는 점이야. HD현대일렉트릭은 2023년 7월 이미 52주 저점 대비 3배 이상 올랐지만, 이후에도 큰 상승이 남아 있었다. 2023년 7월 5일 리포트는 현재가 67,300원, 52주 저가 22,050원을 제시했고, “실적흐름도 밸류에이션도 아직 갈 길이 멀다”고 봤다. ([Naver][10])

## 6-3. Financial Agent

역할은 EPS 폭발 여부를 숫자로 잡는 것.

핵심 계산:

```text
OP_growth_FY2 = FY2_OP / TTM_OP
EPS_growth_FY2 = FY2_EPS / TTM_EPS
OP_delta_mcap = (FY2_OP - TTM_OP) / current_market_cap
OPM_delta = FY2_OPM - TTM_OPM
ROE_delta = FY2_ROE - TTM_ROE
Forward_PER = current_price / FY2_EPS
Forward_PBR = current_price / FY2_BPS
ROE_to_PBR = FY2_ROE / current_PBR
```

적자에서 흑자 전환하는 종목은 EPS 증가율이 무한대처럼 튀므로, 이때는 이렇게 봐야 해.

```text
Turnaround Score =
  FY2_OP / market_cap
+ FY2_OPM
+ FY2_ROE
+ cashflow_improvement
+ debt_risk_penalty
```

하닉 2023년 같은 종목이 여기에 해당해. 적자라서 PER로는 안 보이지만, 다음 EPS가 폭발할 수 있었던 구조야.

## 6-4. Consensus Agent

역할은 “시장이 숫자를 인정하기 시작했는가”를 잡는 것.

계산값:

```text
EPS_revision_1w
EPS_revision_1m
EPS_revision_3m
OP_revision_1w
OP_revision_1m
OP_revision_3m
target_price_revision_1m
analyst_count_change
new_coverage_count
consensus_dispersion
```

좋은 신호:

```text
- 1개월 EPS 컨센서스 +10% 이상
- 3개월 EPS 컨센서스 +20% 이상
- 영업이익 컨센서스 상향
- 목표주가 상향
- 신규 커버리지 증가
- 리포트 제목이 업종 재분류를 시사
```

나쁜 신호:

```text
- 주가는 급등했는데 EPS 컨센서스는 정체
- 목표주가만 오르고 EPS는 안 오름
- 애널리스트 수가 1명뿐인데 컨센서스처럼 보임
- EPS 상향 후 바로 하향 전환
```

## 6-5. Disclosure Agent

공시에서 뽑아야 할 항목은 정해져 있어.

```text
- 단일판매·공급계약
- 신규시설투자
- 생산능력 증설
- 영업실적 공정공시
- 잠정실적
- 장래사업·경영계획
- 유상증자
- CB/BW 발행
- 자기주식 취득/소각
- 최대주주 변경
- 소송/우발채무
```

E2R에서 좋은 공시:

```text
- 수주금액 / 전년 매출액 ≥ 10%
- CAPA 증가율 ≥ 20%
- 신규시설투자금액 / 시총 ≥ 5~10%
- 고마진 제품 증설
- 미국/북미/중동/유럽 등 고마진 수출지역 확대
- 고객사 장기계약
```

위험 공시:

```text
- 유상증자
- 전환사채 반복 발행
- 소송
- 횡령·배임
- 거래정지
- 불성실공시
- 관리종목
```

일진전기의 경우, 2023년 9월 8일 신규시설투자 공시가 1차 레이더 신호였고, 2024년 3월 27일 SK증권 리포트는 초고압변압기 수요 대응을 위한 680억 원 규모 신규 공장과 CAPA 증가를 핵심 투자포인트로 설명했다. ([Naver][11])

```json
{

  "fy2_op": null,
  "fy1_eps": null,
  "fy2_eps": null,
  "investment_points": [
    "수주 증가",
    "이익률 증가",
    "북미 전력기기 호황"
```

```text
전력망
```

```text
Theme_Regime_Score =
  0.30 × high_quality_news_count
```

## 6-8. Rerating Thesis Agent

이 에이전트가 제일 중요해.
단순 숫자 좋은 기업과 리레이팅 기업을 구분한다.

리레이팅 문장은 이렇게 만들어야 해.

```text
기존 분류:
- 전력기기주
- 경기민감 산업재
- 중공업 복합기업

새 분류:
- AI 데이터센터 전력 병목주
- 북미 전력망 교체 수혜주
- 변압기 공급 부족 프리미엄 기업
```

리레이팅 판단 질문:

```text
1. 이 회사가 기존 업종 평균 PER/PBR보다 높은 배수를 받을 이유가 생겼는가?
2. 그 이유가 단순 뉴스가 아니라 EPS로 내려오는가?
3. 고객이 비싸도 살 수밖에 없는 병목 제품인가?
4. 공급자가 제한되어 있는가?
5. 수주잔고나 장기계약으로 이익 지속성이 보이는가?
6. 마진이 개선되고 있는가?
7. 시장이 아직 그 변화를 완전히 반영하지 않았는가?
```

## 6-9. Red Team Agent

이 에이전트가 없으면 시스템이 테마에 취한다.

반드시 반대 논리를 찾아야 해.

```text
- EPS 증가가 일회성인가?
- 영업이익률 개선이 환율 때문인가?
- 수주잔고가 줄고 있는가?
- 신규 수주가 감소했는가?
- 고객사가 한 곳에 집중되어 있는가?
- 주가가 EPS보다 먼저 과도하게 올랐는가?
- PER/PBR이 이미 업종 최고 수준인가?
- 유상증자나 CB 가능성이 있는가?
- 재고가 쌓이는가?
- 매출채권이 급증하는가?
- 현금흐름이 순이익을 따라오지 못하는가?
```

제룡전기가 좋은 예야. 2024년 3월에는 매출 +113.7%, 영업이익 +338.9%, OPM 38.1%, 수출 비중 80%, 수주잔고 +88%라는 강한 E2R 신호가 있었다. ([네이버 프리미엄 콘텐츠][13])
그런데 2025년에는 매출이 직전년도 대비 약 14.7% 감소하고, 영업이익도 관세 등의 영향으로 감소했다는 사업보고서 내용이 나왔다. 이때 Red Team Agent가 등급을 내려야 한다. ([KIND][14])

## EPS Explosion Score

```text
EPS_Explosion_Score =
  0.30 × OP_growth_score
+ 0.20 × EPS_growth_score
+ 0.20 × OP_delta_to_mcap_score
+ 0.15 × OPM_expansion_score
+ 0.10 × ROE_expansion_score
```

| 조건                       |     점수 |

| FY2 OP / TTM OP ≥ 3배     |  75~90 |

| OPM +5%p 이상 개선           |  강한 가점 |

## Valuation Score

```text
Valuation_Score =
  0.30 × forward_PER_attractiveness
+ 0.25 × ROE_to_PBR
+ 0.20 × OP_delta_to_mcap
+ 0.15 × peer_discount
+ 0.10 × PEG_inverse
```

| 방산       | 수주잔고, 장기계약, PER             |

| 금융       | PBR, ROE, 배당, 자사주           |
| 적자 턴어라운드 | OP 증가분/시총, 흑자전환 후 OPM       |

```text
Revision_Score =
```

## Rerating Thesis Score

LLM이 0~100점으로 평가한다.

| 점수     | 의미                                |
| ------ | --------------------------------- |
| 90~100 | 기존 업종에서 구조적 병목 기업으로 재분류 가능성이 매우 큼 |
| 75~90  | 리레이팅 논리 강함, 숫자도 동행                |
| 60~75  | 스토리는 있으나 숫자 일부 미확인                |
| 45~60  | 테마는 있으나 실적 연결 약함                  |
| 45 미만  | 단순 테마주 가능성 높음                     |

## Price Stage Score

```text
if price_6m_return < 30% and EPS_revision_3m > 20%:
    score = 100

elif 30% <= price_6m_return < 100% and EPS_revision_3m > 20%:
    score = 85

elif 100% <= price_6m_return < 250% and EPS_revision_3m > 30%:
    score = 70

elif price_12m_return > 300% and valuation_still_cheap:
    score = 55

elif price_12m_return > 300% and valuation_expensive:
    score = 25

elif price_up_but_eps_flat:
    score = 10
```

이게 중요한 이유는 HD현대일렉트릭 때문이다. 2023년 7월 이미 많이 올랐지만, 실적과 밸류가 더 열려 있었다. 그래서 단순 상승률만으로 탈락시키면 대어를 놓친다.

# 8. 1차·2차·3차·4차 분류 기준

이제 핵심인 “에이전트가 봤을 신호 1, 2, 3, 4차”를 정확히 정의해보자.

---

## 1차: 레이더 진입

**의미:**
아직 매수 후보는 아니지만, 산업 또는 기업에서 변화의 냄새가 난다.

**데이터 출처:**

```text
- 글로벌 뉴스
- 산업 리포트
- DART/KIND 공시
- 신규시설투자
- 수주 공시
- 섹터 주가 상대강도
- 증권사 첫 언급 리포트
```

**조건:**

```text
Theme_Regime_Score ≥ 70
또는
Company_Event_Score ≥ 60
```

**대표 신호:**

```text
- 산업 공급 부족
- ASP 상승 가능성
- 수주잔고 증가
- 신규시설투자
- 고객사 증설
- 고부가 제품 전환
- 북미/미국/유럽/중동 수출 증가
- 데이터센터, 방산, 조선, K-뷰티 같은 구조 성장 키워드
```

**예시:**
일진전기는 2023년 9월 8일 신규시설투자 공시가 1차 레이더 신호였다. 초고압변압기 수요 증가 대응, 680억 원 규모 신규 공장, 변압기 CAPA 증가가 확인됐기 때문이다. ([Naver][11])

---

## 2차: 후보 편입

**의미:**
숫자와 스토리가 같이 보이기 시작한다. 아직 S급은 아니지만 매일 추적해야 한다.

**데이터 출처:**

```text
- FnGuide/QuantiWise 컨센서스
- 증권사 리포트
- DART 분기보고서/사업보고서
- KRX 주가
- 수주잔고/신규수주
- CAPA 증설 공시
```

**조건:**

```text
E2R_Total_Score ≥ 65
EPS_Explosion_Score ≥ 60
Valuation_Score ≥ 50
Rerating_Thesis_Score ≥ 60
```

**정량 기준 예시:**

```text
FY2 OP ≥ TTM OP × 2
또는 적자 → 흑자 전환

OP 증가분 / 시총 ≥ 10%

OPM 개선 ≥ 2%p

FY2 PER이 업종 평균 대비 과도하지 않음

PBR 대비 ROE가 매력적

수주잔고 / 매출 ≥ 0.8배

최근 6개월 주가 상승률이 크더라도 EPS 상향이 동행
```

**예시:**
효성중공업은 2023년 5월 2일 리포트에서 1Q23 부진이 저마진·역마진 수주 정리 때문이며, 1H23 내 마무리 후 중공업 부문 이익률 회복이 예상된다는 논리가 제시됐다. 이건 실적표는 더러운데 다음 이익 체력이 좋아질 수 있다는 2차 후보 신호다. ([Naver][15])

---

## 3차: 고확신 편입

**의미:**
실적, 수주, 컨센서스, 멀티플이 같이 움직인다. E2R의 핵심 구간이다.

**데이터 출처:**

```text
- 분기 실적 발표
- 컨센서스 상향
- 목표주가 상향
- 적용 PER/PBR 상향
- 수주잔고 증가
- ASP 상승
- 공급 부족 확인
- 고객사 장기계약
```

**조건:**

```text
E2R_Total_Score ≥ 80
EPS_Explosion_Score ≥ 75
Revision_Score ≥ 65
Rerating_Thesis_Score ≥ 75
Risk_Penalty 낮음
```

**고확신 트리거:**

```text
- 영업이익이 컨센서스 대비 +10~15% 이상 상회
- FY1/FY2 EPS가 1개월 내 +15% 이상 상향
- 목표주가 +20% 이상 상향
- 리포트에서 적용 PER/PBR 상향
- OPM이 전년 대비 +3%p 이상 개선
- 수주잔고가 사상 최대 또는 매출 1년치 이상
- 공급 부족/리드타임/ASP 상승이 확인
```

**예시:**
HD현대일렉트릭은 2023년 7월 27일 리포트에서 이익률 증가, 끝없이 쌓이는 수주, 목표주가 21% 상향이 동시에 나왔다. 이건 3차 고확신 신호다. ([Naver][12])

LS ELECTRIC도 2024년 4월 30일 리포트에서 1Q24 서프라이즈, 장기 업황 호황 기대, 목표주가 27% 상향, 현재가 173,300원과 52주 저가 61,200원이 제시됐다. 이건 2차 후보에서 3차로 올라갈 수 있는 신호다. ([Naver][16])

## 4차: 졸업생 또는 피크아웃 감시

**의미:**
좋은 기업이지만 초입 발굴 구간은 아니다. 여기서는 “추세 지속”과 “피크아웃”을 구분해야 한다.

**데이터 출처:**

```text
- 주가 상승률
- forward PER/PBR
- 컨센서스 상향 둔화
- 수주잔고 감소
- 신규수주 감소
- OPM 하락
- 투자주의/경고
- 리포트 문구 변화
```

**졸업생 조건:**

```text
12개월 상승률 ≥ 300%
그리고
Forward PER ≥ 35~50배
또는
PBR ≥ 8~10배
그리고
EPS 상향 속도 둔화
```

**피크아웃 조건:**

```text
매출 성장률 둔화
영업이익률 하락
신규수주 감소
수주잔고 감소
컨센서스 하향
재고 증가
매출채권 증가
관세/경쟁/가격 하락 언급 증가
```

**예시:**
제룡전기는 2024년 초에는 강한 E2R 후보였지만, 2025년 실적에서 매출과 영업이익이 감소하면서 4차 피크아웃 감시로 내려가야 했다. ([KIND][14])

대한전선은 2026년 4월 14일 리포트에서 1Q26 컨센서스 상회, 고수익 프로젝트 반영, 12개월 절대수익률 +183.7%가 제시됐지만, E2R 관점에서는 전력망 모멘텀은 강해도 밸류 부담을 따로 체크해야 하는 종목이다. ([AlphaSquare][17])

# 9. “인식 시점”을 어떻게 결정하나

인식 시점은 감으로 정하면 안 돼.
시스템에서는 이렇게 정의해야 해.

```text
인지일 =
그 종목이 해당 날짜의 장전 브리핑에서
처음으로 Stage 2 이상 또는 Stage 3 이상 조건을 충족한 날짜
```

단, 데이터 공개 시간에 따라 다르게 처리한다.

```text
공시가 장중 발표됨:
- 장중 알림 가능
- 장전 브리핑에는 다음 거래일 반영

리포트가 오전 장전 발간됨:
- 당일 장전 브리핑 반영

리포트가 장중/장후 발간됨:
- 다음 거래일 장전 브리핑 반영

컨센서스가 익일 오전 7:30경 업데이트됨:
- 그날 장전 브리핑에 반영
```

즉 백테스트에서는 반드시 `published_at`, `available_at`, `briefing_date`를 분리해야 한다.

# 11. 실제 단계 분류표

```text
Stage 0: 무시
- 테마도 없고 숫자도 없음

Stage 1: 레이더
- 섹터 레짐 변화 또는 기업 이벤트 발생
- 아직 EPS/밸류 확신은 부족

Stage 2: 후보 편입
- EPS/OP 증가 가능성 확인
- 밸류가 아직 감당 가능
- 수주/CAPA/ASP/고부가 믹스 중 하나 이상 확인

Stage 3: 고확신
- 실적 서프라이즈
- 컨센서스 상향
- 목표주가 또는 적용 멀티플 상향
- 수주잔고/ASP/마진 개선 확인
- 리레이팅 서사 명확

Stage 4: 졸업생/피크아웃 감시
- 이미 큰 폭 상승
- 밸류 부담
- EPS 상향 둔화
- 수주·마진 둔화
- 그래도 추세는 유지될 수 있으나 신규 발굴 구간은 아님
```

등급은 이렇게 붙이면 돼.

| 등급   | 기준                                   |
| ---- | ------------------------------------ |
| S    | Stage 3, 총점 85 이상, EPS·밸류·리레이팅 모두 강함 |
| A    | Stage 2~3, 총점 75~84, 구조 강하지만 일부 리스크  |
| B    | Stage 2, 총점 60~74, 방향은 맞지만 숫자 일부 미확인 |
| C    | Stage 1, 총점 45~59, 테마는 있으나 EPS 약함    |
| 졸업생  | 좋은 기업이지만 가격·밸류가 이미 많이 반영             |
| 피크아웃 | 수주·마진·컨센서스 둔화 시작                     |
| 제외   | 재무위험, 공시위험, 유동성 부족, 테마만 존재           |

# 16. 가장 중요한 백테스트 원칙

이 시스템을 만들 때 제일 위험한 건 **후견지명**이야.

하면 안 되는 것:

```text
- 2026년 실적을 보고 2023년에 잡았다고 착각
- 현재 컨센서스를 과거 날짜에 적용
- 상장폐지/관리종목을 유니버스에서 누락
- 이미 성공한 종목만 훈련
- 리포트 발간일과 실제 접근 가능 시점 혼동
```

반드시 해야 하는 것:

```text
- 모든 데이터에 as_of_date 저장
- 리포트 published_at 저장
- 공시 접수시간 저장
- 컨센서스 업데이트 날짜 저장
- 가격은 전일 종가 기준으로 계산
- 실패 사례도 함께 학습
```

# 18. 최종 압축

매일 아침 브리핑 에이전트를 만들려면 핵심은 이거야.

```text
1. KRX/KIS로 가격과 유동성 확인
2. OpenDART/KIND로 공시와 리스크 확인
3. FnGuide/QuantiWise/WiseReport로 EPS 컨센서스와 상향률 확인
4. 증권사 리포트에서 목표주가, 적용 멀티플, 수주·마진 논리 추출
5. 뉴스/API로 산업 레짐 변화 확인
6. Quant Score로 전 종목 압축
7. LLM Agent로 리레이팅 서사 검증
8. Red Team Agent로 테마 과열과 피크아웃 제거
9. Stage 1~4로 분류
10. 신규 편입·상향·하향만 매일 브리핑
```
