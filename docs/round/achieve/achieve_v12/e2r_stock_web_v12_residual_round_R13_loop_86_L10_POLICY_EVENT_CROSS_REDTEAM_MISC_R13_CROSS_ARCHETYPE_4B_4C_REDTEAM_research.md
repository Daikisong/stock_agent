## 4. Research question

Loop86 deliberately tested many sector-specific “theme vs bridge” pairs:

```text
EPC contract vs margin/cash bridge
HBM equipment beta vs fresh order/customer bridge
battery contract headline vs call-off/shipment bridge
chemical beta vs spread/margin bridge
consumer food theme vs export reorder bridge
insurance beta vs reserve/capital-return bridge
medical-device export theme vs reimbursement/channel bridge
AI/security theme vs contract-retention/ARR bridge
mobility theme vs volume/mix/margin bridge
construction/PF theme vs liquidity/backlog/cash bridge
defense/geopolitical theme vs export-framework backlog bridge
holdco governance theme vs direct tender/control-premium/capital bridge
```

The shared failure mode is simple:

```text
theme label arrives first
business bridge is missing or proxy-only
MFE is small or unstable
MAE opens wide
```

R13 tests this diagnostic guard:

```text
Hard 4B guard:
bridge_missing_or_unverified
AND (
  (MFE90 < 10 AND MAE90 <= -20)
  OR
  (MFE90 < 20 AND MAE180 <= -25)
)
=> block Stage2-Actionable / Yellow / Green
=> route to Watch / 4B-risk / evidence-quality repair
```

Soft watch guard:

```text
bridge_missing_or_unverified
AND MFE90 < 5
=> block Yellow / Green until exact bridge evidence is repaired
```

Positive-control exception:

```text
If canonical bridge is present and MFE90 >= 20,
preserve Stage2/Yellow eligibility.
If MAE90 <= -20 even with bridge present,
do not block Stage2 automatically, but keep Green strict and require exact evidence.
```
