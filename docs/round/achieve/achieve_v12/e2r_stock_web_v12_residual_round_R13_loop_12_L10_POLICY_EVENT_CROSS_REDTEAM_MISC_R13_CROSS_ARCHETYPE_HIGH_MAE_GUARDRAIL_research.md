## 1. Selection rationale

The latest no-repeat index still shows static Priority 0 and Priority 1 gaps, but the conversation-local ledger now contains repeated final-pass or static-to-50 cleanup files for the major C01~C32 groups. At this point the highest marginal value is not another sector floor-fill. It is a cross-archetype checkpoint that asks whether rows with **positive MFE, local 4B behavior, or apparent Stage2/Stage3 setup** are still too dangerous because the 180D drawdown is large and the evidence remains `source_proxy_only`.

This file therefore uses the R13 scope:

```text
R13_CROSS_ARCHETYPE_HIGH_MAE_GUARDRAIL
```

The guardrail target is simple: if a row has a visible upside path but also deep 90D/180D MAE, it behaves like a car that can accelerate but has weak brakes. It may be useful for watch/retest logic, but it should not be allowed to become Stage3-Green or full 4B until the bridge is reverified.
