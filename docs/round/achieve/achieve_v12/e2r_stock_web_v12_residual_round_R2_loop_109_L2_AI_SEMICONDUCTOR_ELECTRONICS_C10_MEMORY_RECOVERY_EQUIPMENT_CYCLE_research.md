## 1. Selection rationale

The latest V12 index still lists `C10_MEMORY_RECOVERY_EQUIPMENT_CYCLE` in Priority 1 with 37 representative rows and 13 rows needed to reach the 50-row operating-calibration band. Recent local files have pushed C01, C06, C07, C11 and several Priority 0 families toward local floors, while C10 has no dedicated local residual file in the current `/mnt/data` ledger. This run therefore isolates rows previously sitting near C06/C07 as contaminants or boundary cases and reinterprets them under C10.

C10 is not “HBM equipment order relative strength” and not “memory customer capacity.” The mechanism is more like a tide lifting tools before ship manifests arrive: memory price/capex recovery pulls equipment names up, but the signal is only durable when it travels into order visibility, revenue recognition, margin recovery and FCF. A price-only rally is only surf foam; C10 needs the tide table underneath.
