## 13. Current Calibrated Profile Stress Test
The residual error is not that Stage2 is early in general; it is that C10 has many “route/proxy” rows where the sector weather is strong but issuer conversion is still missing. The current proxy can still over-credit product/facility/material route as if it were booked order or shipment conversion.

## 14. Stage2 / Yellow / Green Comparison
- Stage2-Actionable works for `045100` where the route is issuer-specific and drawdown is controlled.
- Stage3-Yellow works for `058470` and `281740` because the route is direct and MFE confirms upside, but deep full-window MAE argues against Green.
- Stage3-Green is only defensible in this sample for `317330`, where entry was early/reset and MAE stayed contained through the key window.
- Generic proxy rows (`140070`, `093370`, `220260`) should not inherit Green-like treatment.

## 15. 4B Local vs Full-window Timing Audit
Rows with local spike and later deep drawdown are treated as 4B-watch rather than hard 4C. `025560`, `058470`, and `281740` show that live route plus high MAE is a phase-risk problem, not automatic thesis death.

## 16. 4C Protection Audit
No hard 4C row is proposed. The evidence does not prove customer/order route death; it proves that proxy evidence and late/local spike entries need a watch cap.

## 17. Sector-Specific Rule Candidate
L2 memory/electronics rows should split memory-cycle weather, issuer-specific product/facility/material route, customer qualification, booked order or shipment conversion, margin/cash bridge, and price phase. The memory upcycle is wind; only issuer-level orders turn the turbine.

## 18. Canonical-Archetype Rule Candidate
C10 should use a proxy-to-conversion freshness ladder: generic memory/HBM/facility/material cycle → issuer product route → customer qualification/order → shipment/utilization → margin/cash conversion. Route evidence can be Actionable or Yellow after reset, but Green waits for conversion; generic proxy after rerating becomes 4B-watch.
