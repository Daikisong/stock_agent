## 5. Cross-Archetype Finding
The common failure is not absence of Stage2 evidence. Most rows had a real public trigger: customer, product, order, contract, approval, capacity, or recovery evidence. The failure appears when Stage2 evidence is promoted as if it already contained shipment, revenue recognition, margin, FCF, renewal, or utilization proof. Once the price path is checked, the weak bridge shows up as high 90D/180D MAE or poor MFE/MAE asymmetry.

The proposed R13 guardrail is:

```text
R13_STAGE2_FALSE_POSITIVE_BRIDGE_GAP_GATE_SECOND_PASS
```

It should cap Stage2-Actionable / Stage3-Yellow promotion into `Stage2-Watch + local 4B overlay` when:

```text
1. evidence is real but mostly headline/proxy/theme/order announcement,
2. shipment/revenue/margin/FCF bridge is not yet confirmed,
3. MAE_90D <= -25% or MAE_180D <= -30%, or
4. MFE_180D < 15% while MAE_180D <= -30%.
```
