### Why C16 now

C16 is the point where a headline can fool the model by wearing the costume of scarcity. The weak path is:

```text
policy/resource/cable keyword -> price spike -> imagined strategic scarcity
```

The stronger, investable path is:

```text
funded offtake / capacity lock / regulated project / customer order -> volume or ASP -> margin -> working-capital absorption -> FCF
```

The purpose of this pass is to keep the second path alive while stopping the first path from leaking into Stage3-Green or full 4B.

## 7. Interpretation

### 7.1 What should score better

A C16 positive should receive credit only when the strategic-resource or supply-chain thesis reaches company cashflow. In plain terms, the policy must leave footprints in the company ledger:

```text
offtake / funded project / capacity lock -> order or utilization -> margin -> FCF
```

Examples in this pass that deserve conditional positive treatment are grid/cable capacity names where MFE was large but MAE remained manageable only when the order/capacity bridge was plausible.

### 7.2 What should be capped

Rows with large MFE but deep MAE are the familiar trap. They are like a flare in a dark field: bright enough to catch the eye, but not warm enough to build a fire. The model should cap them as local 4B or Stage2 watch when the non-price bridge is missing.

C16 should not absorb all of C15, C17, or C02. If the mechanism is mostly commodity spread, reroute to C17. If it is broad material supercycle, reroute to C15. If it is datacenter/grid CAPEX backlog rather than strategic resource policy, reroute to C02.
