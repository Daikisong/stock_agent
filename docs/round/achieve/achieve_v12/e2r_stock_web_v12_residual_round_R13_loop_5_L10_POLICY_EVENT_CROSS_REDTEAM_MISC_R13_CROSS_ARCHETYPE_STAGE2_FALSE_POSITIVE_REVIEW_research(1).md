## 1. Selection rationale

The latest run uses the V12 coverage-index scheduler rather than a mechanical R1→R13 cycle. The local ledger already contains final-pass or near-final-pass artifacts for many Priority 0 and Priority 1 sectors. After those floor-fill loops, the highest-value next residual is not another sector row but a cross-archetype checkpoint: Stage2 and Stage2-Actionable rows that looked alive on MFE, but carried too much MAE, too little verified non-price bridge, or the wrong canonical route.

This file therefore uses `R13_CROSS_ARCHETYPE_STAGE2_FALSE_POSITIVE_REVIEW`. R13 is not a sector. It is the customs gate at the end of the conveyor belt: before a row enters a profile patch, it checks whether the row is really a signal or just a sector label wearing a signal's clothes.

## 3. Research question

A Stage2 row should be the first footprint in wet concrete, not the finished building. The residual error is that some Stage2 rows receive enough narrative weight to behave like Stage3 candidates even when the company-level bridge is missing.

R13 checks five repeated failure modes:

```text
1. sector keyword but no company cash bridge
2. Stage2-Actionable bonus applied to source_proxy_only evidence
3. positive 30D/90D MFE but deep 180D MAE
4. wrong canonical route, e.g. battery orderbook vs AMPC vs EV demand slowdown
5. reused price row or evidence_url_pending row proposed for profile promotion too early
```
