## 3. Core finding
R13 accounting/trust/price validation should act like a customs gate between *event evidence* and *stage persistence*. A row may have a real contract, policy, customer, margin, or demand headline; however, if the second bridge is missing, Stage2/Stage3 persistence can still be wrong. The recurring failure path is:

```text
valid headline evidence
→ missing cash collection / revenue recognition / OPM·FCF / delivery / execution bridge
→ temporary MFE or theme premium
→ deep 90D/180D MAE
→ required Stage4B-LocalWatch overlay until bridge confirmation
```
