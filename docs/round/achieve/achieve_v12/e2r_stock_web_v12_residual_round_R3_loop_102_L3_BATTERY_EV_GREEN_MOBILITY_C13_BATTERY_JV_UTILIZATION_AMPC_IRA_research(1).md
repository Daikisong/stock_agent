## 13. Current Calibrated Profile Stress Test

C13 residual problem is not that the calibrated profile is globally too strict or globally too loose. The residual is that source-proxy AMPC/JV/localization labels can still be over-read as bridge evidence unless C13 explicitly asks: did utilization, customer pull, delivery acceptance, revenue, margin, or FCF appear? Where that bridge is missing and MAE expands, a local 4B/4C watch is preferred.

## 14. Stage2 / Yellow / Green Comparison

```text
Stage2 watch allowed = policy/localization/customer-pull proxy exists
Stage2-Actionable allowed = verified utilization, customer acceptance, delivery or order conversion evidence appears
Stage3-Yellow allowed = margin or revenue conversion becomes visible
Stage3-Green allowed = durable bridge + revision + low red-team risk; not supported by this source-proxy-only loop
```

## 15. 4B Local vs Full-window Timing Audit

C13 label-led equipment/material names often hit early local peaks and then draw down hard. The proposed handling is not automatic full 4B; it is `local_4B_watch` unless non-price overhang, contract delay, customer pull break, margin break, or utilization failure is also present.

## 16. 4C Protection Audit

Only `079810` is routed as Stage4C in this loop. The rule is weakened against generic label-only 4C: a hard 4C should require confirmed order/call-off/utilization/margin break, not only a fading policy or battery-equipment label.
