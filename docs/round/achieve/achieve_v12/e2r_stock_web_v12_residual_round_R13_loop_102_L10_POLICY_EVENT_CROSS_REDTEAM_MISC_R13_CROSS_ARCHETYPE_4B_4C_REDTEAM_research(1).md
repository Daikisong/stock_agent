## 4. Cross-Archetype Finding

이 표본의 공통점은 단순히 “가격이 많이 빠졌다”가 아니다. 각 row는 모두 sector별로 그럴듯한 비가격 evidence를 갖고 있었다. 전력망/UC 기술, 장비 utilization 회복, 배터리 separator·동박·소재 계약, 플랫폼/NFT event cap, EV 수요둔화, mobility supplier margin break, 반도체 장비 estimate cut 같은 evidence가 있었다. 문제는 **어떤 evidence는 local 4B로 빨리 눌러야 했고, 어떤 evidence는 hard 4C로 확정해야 했으며, 일부는 반대로 4B/4C를 너무 빨리 때리면 살아 있는 thesis를 잘라버린다**는 점이다.

이 R13 pass의 핵심은 세 갈래다.

1. `fast local 4B`: MFE가 열려도 90D/180D MAE가 깊고 delivery/revenue/margin bridge가 약하면 Stage3 승격 전에 local 4B overlay가 필요하다.
2. `hard 4C`: 고객 재고조정, utilization break, 영업손실, thesis break가 확인되면 섹터를 불문하고 hard 4C로 빠르게 route해야 한다.
3. `overkill buffer`: estimate cut, ASP/재고손실, 수요둔화가 있어도 customer/order/technology thesis가 살아 있으면 hard 4C 또는 full 4B를 너무 빨리 적용하면 안 된다.

### 4.1 Rule candidate

```text
R13_4B_4C_REDTEAM_TIMING_SPLIT_LOCAL_4B_HARD_4C_OVERKILL_BUFFER
```

Proposed guardrail:

```text
if candidate_stage in {Stage2-Actionable, Stage3-Yellow, Stage3-Green}:
    if realized_path_has_fast_MFE_but_MAE90 <= -25% or MAE180 <= -40%:
        if confirmed_delivery_revenue_margin_bridge is missing:
            cap to Stage2-Watch + local Stage4B overlay

if non_price_evidence confirms utilization break, order/call-off, operating loss, trust break, or thesis failure:
    route to hard Stage4C regardless of sector theme strength

if negative evidence is estimate cut, ASP lag, inventory loss, or demand slowdown but surviving customer/order/technology evidence remains:
    use Stage4B-Watch rather than hard Stage4C
    require follow-up confirmation before full thesis break
```
