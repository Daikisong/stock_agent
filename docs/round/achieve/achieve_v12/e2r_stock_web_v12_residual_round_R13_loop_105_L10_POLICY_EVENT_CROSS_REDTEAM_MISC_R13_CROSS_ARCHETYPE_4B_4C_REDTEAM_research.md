## 6. Core Finding
The residual error is not that every failed row should be treated as a permanent 4C. The error is subtler: **Stage4B and Stage4C are different brakes**. Local Stage4B is the shock absorber for fast MFE followed by deep MAE when the non-price bridge is missing. Hard Stage4C is the emergency cut-off for explicit asset-level or business-thesis break. In this holdout set, only 신라젠/Pexa-Vec qualifies as hard 4C; the rest require faster local 4B overlays rather than thesis deletion.

## 7. Shadow Rule Candidate
```text
R13_4B_4C_TIMING_SPLIT_HARD_4C_VS_LOCAL_4B_HOLDOUT_V105
```

Suggested behavior: keep `hard_4c_thesis_break_routes_to_4c` strict when explicit trial futility, regulatory rejection, production halt, accounting trust break, or final contract failure exists. Otherwise, if MAE90/MAE180 breaches while shipment/revenue/margin/retention/utilization bridge is still absent, downroute Stage2/Stage3 persistence into `Stage4B-LocalWatch` rather than full 4C.
