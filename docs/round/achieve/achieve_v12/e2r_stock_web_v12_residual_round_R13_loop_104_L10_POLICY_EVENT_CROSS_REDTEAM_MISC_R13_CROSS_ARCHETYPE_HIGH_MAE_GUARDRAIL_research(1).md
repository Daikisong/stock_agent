## 1. Selection Rationale
The main execution prompt requires `coverage_index_first`; R13 is not a normal sector round. R13 is allowed only as a cross-archetype checkpoint for false positives, 4B/4C timing, accounting/trust/price validation, or high-MAE guardrails. The previous local R13 loops covered 4B/4C red-team, Stage2 false-positive, and accounting/trust/price-validation; the remaining distinct R13 scope is therefore `R13_CROSS_ARCHETYPE_HIGH_MAE_GUARDRAIL`.

This loop asks a narrower question:

> When a case has a deep 90D drawdown, should E2R delete the opportunity, keep it as Stage2, or route it to local 4B/4C?

The answer from this holdout is not “MAE is always bad.” It is more surgical: verified non-price bridge plus high MFE can remain Stage2-Actionable with a high-MAE/position-size/4B-watch overlay, but label/proxy rows with low MFE and deep MAE should be blocked from Yellow/Green and routed to local 4B or hard 4C.

## 4. Residual Finding

### 4.1 What current calibrated profile still misses
The existing calibrated profile already blocks price-only blowoff, requires non-price evidence for full 4B, and requires bridge evidence for Stage2. The residual problem is subtler: **path dependency**.

A case can have legitimate evidence and still punish entry with a -25% to -55% MAE before producing MFE. These rows should not become Green merely because the eventual MFE is high. They should remain Stage2-Actionable or Yellow only with explicit high-MAE warnings: staged entry, local 4B watch, and no Green until bridge confirmation survives the drawdown window.

### 4.2 Proposed shadow rule
```text
axis = R13_high_MAE_guardrail_overlay
scope = R13_CROSS_ARCHETYPE_HIGH_MAE_GUARDRAIL
rule_candidate:
  if MAE_90D_pct <= -25:
      add high_MAE_path_risk_overlay
      block Stage3-Green unless verified bridge + recovery confirmation exist
  if MAE_90D_pct <= -50:
      force local_4B_watch unless MFE90>=40 and source bridge is direct/verified
  if MFE_90D_pct < 15 and MAE_90D_pct <= -30:
      block Stage2-Actionable/Stage3-Yellow unless issuer-level thesis bridge is repaired
      route to local_4B_watch or hard_4C depending on non-price thesis break
```

This is not a global threshold relaxation. It is a cross-archetype guardrail: like a seatbelt, it does not decide the destination, but it changes how fast the machine is allowed to move when the road is icy.
