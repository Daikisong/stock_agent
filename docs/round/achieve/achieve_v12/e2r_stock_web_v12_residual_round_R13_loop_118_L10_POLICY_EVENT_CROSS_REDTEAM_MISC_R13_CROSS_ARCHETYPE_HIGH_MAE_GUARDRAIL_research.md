## 2. Core finding

A valid non-price event can still be a bad persistence path. The repeated mechanism is:

```text
valid public event exists
→ shipment / delivery / revenue recognition / margin / FCF / utilization / cash / retention bridge is incomplete
→ MFE opens briefly or unevenly
→ MAE90/MAE180 breaches deeply
→ Stage2/Stage3 persistence should be capped or overlaid by local Stage4B
```

This is not a blanket 4C rule. If an explicit thesis break exists, route to hard 4C. If no hard break exists but MAE is deep, keep the thesis as watch/local 4B rather than letting it persist as clean Stage2/Stage3.

The current profile already contains `stage2_required_bridge` and `local_4b_watch_guard`, but this holdout says the guardrail should be applied more persistently after the first deep MAE breach. A row can have real Stage2 evidence and still need a Stage2-Watch cap when the second bridge confirmation is absent.

Proposed guardrail candidate:

```text

```

Operational form:

```text
If MAE90 <= -25% or MAE180 <= -30%, and no confirmed delivery / revenue recognition / margin / FCF / utilization / cash / retention bridge has appeared, cap the path at Stage2-Watch or force local Stage4B overlay.
```
