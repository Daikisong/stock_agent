## 6. Canonical Archetype Compression Map

C10 should be compressed into a ladder:

```text
generic memory/HBM weather
  -> product route / customer route
  -> direct order, shipment, or capacity conversion
  -> utilization/margin bridge
  -> price phase sanity / 4B watch
```

The defect found here is not that memory recovery is irrelevant. It is that the market treats a broad HBM/CXL headline like a purchase order. The turbine does not spin from wind alone; C10 needs the shaft connecting memory weather to issuer-specific orders.

## 13. Current Calibrated Profile Stress Test

1. Current profile likely promotes generic HBM/CXL proxy rows too quickly when relative strength is high.
2. That judgment worked for QRT and eventually for TLB, but failed for Koh Young, GigaVis, DNF, Hanyang Digitech, and Mirae Semiconductor.
3. Stage2 Actionable bonus is too generous for broad memory weather with no issuer order conversion.
4. Yellow threshold itself is not too low; the missing gate is **what counts as revision/customer conversion**.
5. Green threshold / revision gate should not be loosened for C10 proxy rows.
6. Price-only blowoff guard remains appropriate.
7. Full 4B non-price requirement remains appropriate, but C10 needs a local 4B watch when proxy rows spike without order conversion.
8. Hard 4C routing should not be triggered by high MAE alone when the direct route survives; it should require customer/order route death.

## 14. Stage2 / Yellow / Green Comparison

No Stage3-Green trigger is proposed. `356860` is allowed to move toward Stage3-Yellow because the source path contains named Samsung/SK hynix sample routing; broad HBM headlines remain Stage2-watch or capped Stage2. `405100` and `104830` are positive but high-MAE rows; they support early Stage2-Actionable but not unconditional Green.

## 15. 4B Local vs Full-window Timing Audit

C10 full 4B should require non-price evidence such as order delay, utilization disappointment, or margin downgrade. However, a **local 4B watch** is justified when a proxy row has:

- 30/90D MFE concentrated near the entry period;
- no direct order conversion;
- 90/180D MAE worse than -40%;
- peak date close to trigger date.

Rows `098460`, `420770`, `092070`, `078350`, and `254490` fit this watch pattern.

## 16. 4C Protection Audit

No hard Stage4C row is issued in this loop. High-MAE alone is not thesis death. For C10, 4C should be reserved for confirmed customer/order route death, qualification failure, or repeated conversion failure after direct order visibility.

## 17. Sector-Specific Rule Candidate

`L2_AI_SEMICONDUCTOR_ELECTRONICS` memory-equipment rows should split memory price/capex weather, supplier-specific product route, customer/order conversion, utilization or shipment bridge, margin conversion, and price phase. The memory upcycle is wind; only issuer-level orders turn the turbine.

## 18. Canonical-Archetype Rule Candidate

`C10_MEMORY_RECOVERY_EQUIPMENT_CYCLE` should add an **HBM/CXL order-conversion freshness ladder**: generic memory recovery -> Stage2-watch; product/customer route -> Stage2-Actionable; direct sample/order/shipment plus utilization or margin bridge -> Yellow/Green; broad proxy after local rerating -> 4B-watch; confirmed customer/order route death -> 4C.
