## 4. Residual interpretation

### 4.1 What current profile still misses

The current calibrated profile already blocks generic price-only blowoff and demands non-price evidence for full 4B. The residual C14 problem is subtler:

```text
A battery/EV name can show:
- enough theme momentum to look Stage2/Stage3,
- enough local MFE to avoid immediate rejection,
- but still suffer a 90D/180D path where MAE is deeper than the useful signal.
```

For C14, that pattern is not just volatility. It is often the market discovering that backlog/orderbook/capex words did not convert into customer call-off, utilization, margin, and FCF.

### 4.2 C14-specific rule candidate

```text
C14_EV_DEMAND_TO_UTILIZATION_CALLOFF_MARGIN_BRIDGE_REQUIRED

If EV demand slowdown / inventory destocking / customer call-off delay is present,
then Stage3-Green is blocked unless at least one of the following refreshes:
1. customer call-off or shipment run-rate,
2. factory utilization or acceptance/payment schedule,
3. ASP/mix/margin bridge,
4. FCF or working-capital bridge.
```

### 4.3 Stage4C hard break candidate

```text
C14_HARD_4C_IF_MAE180_LT_MINUS40_AND_NO_NON_PRICE_BRIDGE

If MAE180 <= -40% and no fresh non-price bridge appears within 90D,
route the case to Stage4C or hard cap below Stage3-Yellow.
```

This is not a universal drawdown rule. It applies only when the mechanism is demand/call-off/utilization, not when a separate verified customer qualification or margin bridge is present.

### 4.4 Counterexample guard

The counterexamples matter. Some names still show meaningful MFE when customer qualification, mix recovery, or margin bridge refreshes before the full 180D pain. The rule should not turn C14 into a blind “battery bad” label.

```text
C14_COUNTEREXAMPLE_GREEN_ALLOWED_WITH_CUSTOMER_QUALIFICATION_OR_MIX_RECOVERY

Permit Stage3-Yellow/Green only when demand-slowdown evidence is offset by
a verified customer/mix/utilization/cash bridge, not by theme rebound alone.
```
