## 1. Required first-read / selection basis

`docs/core/V12_Research_No_Repeat_Index.md` 기준으로 현재 corpus는 단순 row-fill 단계가 아니다. C01~C32는 모두 80 row 이상이고, 대표 row는 11,200개다. 이번 실행은 직전 C01/C05/C10/C15 반복을 피하면서 C13의 URL/proxy/MFE-MAE 품질 보강과 finality → utilization → cashflow ladder를 보강한다. Local archive 기준 C13의 최대 loop는 235였으므로 이번 식별자는 loop 236이다.

## 6. Residual profile stress test

현재 profile의 잔여 오류는 네 갈래다.

1. **Equipment route를 utilization처럼 읽는 오류**: `056190`, `290670`, `251630`, `302430`, `262260`은 sector report의 process/backlog proxy는 있었지만 issuer-level order/cash bridge가 부족했고, 90D~180D MAE가 깊었다.

2. **Direct order라도 phase가 앞서 달리면 Green이 아닌 오류**: `282880`은 KRW 32bn PO가 있었지만 30D/90D MAE가 깊어 Stage3-Green이 아니라 Yellow + local 4B-watch가 맞다.

3. **Reset 이후 direct route는 miss될 수 있는 오류**: `137400`, `222080`은 reset 이후 order/revenue recognition bridge가 살아 있어 Stage2-watch에 묶으면 too-late가 된다. 다만 Green은 utilization/cash가 보일 때까지 보류한다.

4. **High-MAE를 곧장 hard 4C로 태우는 오류**: deep-MAE row라도 customer/order route가 살아 있으면 thesis death가 아니라 4B-watch다.

## 7. Sector-specific rule candidate

`L3_BATTERY_EV_GREEN_MOBILITY`에서는 product-route / equipment-capability headline, customer/order finality, calloff/utilization, ex-credit margin/cash conversion, and price phase를 분리해야 한다. 배터리 장비 route는 배관이다. Green promotion은 배관이 있다는 말이 아니라 실제 물이 흐른다는 증거가 필요하다.

## 8. Canonical archetype rule candidate

`C13_BATTERY_JV_UTILIZATION_AMPC_IRA` should use a **battery-equipment finality-to-utilization ladder**:

```text
product route / equipment capability / sector backlog proxy
→ named customer / direct order / purchase-order finality
→ calloff / shipment / installation / utilization
→ ex-credit margin / operating cash conversion
→ price-phase sanity
```
Stage2는 route evidence에서 시작할 수 있다. Stage2-Actionable은 고객·주문 bridge가 필요하다. Stage3-Yellow는 finality와 가격 phase가 맞을 때 허용한다. Stage3-Green은 utilization/cash conversion이 보여야 한다. High-MAE with surviving route는 hard 4C가 아니라 4B-watch다.
