## Human Synthesis

C10 behaves like a turbine in wind. The memory upcycle is the wind, but it only becomes company earnings if the specific company owns the blade, receives the order, ships the tool, and converts it into margin or cash. This run separates four layers:

1. **Generic recovery weather**: HBM/CXL/AI memory demand exists, but no issuer-level order route. This is Stage2-watch only.
2. **Product or qualification route**: cube prober, CXL tester, scrubber/chiller, socket, CMP equipment, or reflow tool has a real technical route. This can become Stage2-Actionable.
3. **Booked order / shipment conversion**: named customer order, first shipment, or explicit mass-production tester adoption. This can move to Yellow/Green, but only if not already at local rerating exhaustion.
4. **Margin/cash conversion**: realized margin, backlog conversion, repeat order, or utilization proof. This is the missing link in most false positives.

Positive rows are Techwing, STI, and Hanmi: product-route or direct order evidence appeared early enough that 90D/180D MFE dominated. Counterexample rows are Neosem, Unisem, TSE, KCTech, and YC: the text was real, but either it was too proxy-like or it arrived after local rerating, so MAE overwhelmed the thesis before the bridge could mature.

## Current Profile Stress Test

- `stage2_required_bridge`: strengthened. A bridge must mean more than a theme word; it should be product/customer/order/shipment or margin/cash evidence.
- `local_4b_watch_guard`: strengthened. Direct order after a local rerating should be watch/position-size capped rather than treated as a fresh Green.
- `full_4b_requires_non_price_evidence`: kept. Price-only drawdown is not enough; but high MAE plus stale result/proxy text should cap promotion.
- `hard_4c_thesis_break_routes_to_4c`: qualified. High MAE alone should not be hard 4C when customer/product route survives.
