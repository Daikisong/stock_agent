## 4. Research question

Loop88 deliberately tested many sector-specific “theme vs bridge” pairs:

```text
power-grid theme vs order/customer/margin bridge
advanced equipment valuation theme vs customer/order/shipment bridge
EV/battery rebound vs demand/call-off/utilization bridge
material spread theme vs inventory/ASP/margin/cash bridge
beauty/global distribution theme vs reorder/sell-through/margin bridge
insurance distribution beta vs reserve/capital-return bridge
bio clinical/regulatory theme vs approval/commercial/revenue bridge
ad-tech/platform theme vs retention/billing/margin bridge
EV auto-parts spike vs fresh volume/mix/margin bridge
construction rebound vs PF/liquidity/backlog/cash bridge
civil works/EPC theme vs project-scope/margin/cash bridge
governance/holdco theme vs control-premium/capital-return/minority-protection bridge
```

The shared failure mode remains:

```text
theme label arrives first
business bridge is missing or proxy-only
MFE is small, unstable, or local price-only
MAE opens wide
late rebound tries to rewrite the original weak entry
```

R13 loop88 tests this diagnostic guard:

```text
Hard 4B guard:
bridge_missing_or_unverified
AND (
  (MFE90 < 10 AND MAE90 <= -20)
  OR
  (MFE90 < 20 AND MAE180 <= -25)
)
=> block Stage2-Actionable / Yellow / Green
=> route to Watch / 4B-risk / evidence-quality repair
```

A second price-only blowoff guard is retained:

```text
Price-only event blowoff guard:
bridge_missing_or_unverified
AND MFE90 >= 20
AND MAE90 <= -20
=> do not count MFE as positive proof
=> route to 4B-watch unless exact non-price bridge is repaired
```

Loop88 also adds a softer governance event guard:

```text
Governance/holdco local guard:
C32 governance bridge missing
AND MAE90 <= -12
=> local MFE or split/restructuring theme is not enough
=> require tender/capital-return/minority-protection bridge before Yellow
```

Positive-control exception:

```text
If canonical bridge is present and MFE90 >= 20,
preserve Stage2/Yellow eligibility.
If MAE180<=-20 even with bridge present,
do not block Stage2 automatically, but keep Green strict and require exact evidence.
```

C14 risk-protection exception:

```text
If the row is explicitly a 4B/4C risk-protection positive,
do not treat it as a buy-positive.
Count it as evidence that Yellow/Green should be blocked under demand/call-off/utilization slowdown.
```
