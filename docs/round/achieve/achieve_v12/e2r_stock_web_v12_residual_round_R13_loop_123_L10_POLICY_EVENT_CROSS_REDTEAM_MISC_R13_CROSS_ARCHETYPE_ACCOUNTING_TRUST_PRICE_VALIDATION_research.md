## 3. Core finding

Accounting/trust validation is not only about fraud or restatement. In this corpus the repeated failure mode is more mundane: a valid contract, approval, policy, customer, or order headline exists, but the financial bridge is not yet proven. R13 should therefore force an accounting/trust checkpoint whenever any of the following are missing after the first Stage2 event:

```text
cash collection
revenue recognition
shipment or delivery confirmation
margin / OPM / FCF bridge
utilization or call-off clearance
reserve / working-capital / receivables quality
listed-parent materiality
```

When the second bridge is missing and the price path breaches MAE90 or MAE180, the row should not persist as Stage2-Actionable or Stage3-Yellow. It should receive Stage2-Watch or local Stage4B overlay unless the source row has a confirmed hard 4C thesis break.
