## 1. Selection rationale

`V12_Research_No_Repeat_Index.md` keeps `C11_BATTERY_ORDERBOOK_RERATING` in Priority 1 with static rows = 32 and need-to-50 = 18. After the recent Priority 0 local-floor work, C11 is the next high-value Priority 1 bucket: it sits between the bullish battery orderbook family and the C12/C13/C14 guardrail families. The mechanism is simple: an orderbook is only valuable when it moves through the gears of call-off, utilization, ASP/mix, margin, and cash conversion. If one gear slips, the price path often looks like a short squeeze rather than a rerating.

This file is therefore a Priority 1 expansion to the 50-row operating-calibration band. It reuses battery/EV stock-web price rows already verified in adjacent C12/C13/C14 local V12 files, then remaps them into C11 only when the event family is semantically an orderbook rerating or orderbook false-positive path. It does **not** change production scoring.
