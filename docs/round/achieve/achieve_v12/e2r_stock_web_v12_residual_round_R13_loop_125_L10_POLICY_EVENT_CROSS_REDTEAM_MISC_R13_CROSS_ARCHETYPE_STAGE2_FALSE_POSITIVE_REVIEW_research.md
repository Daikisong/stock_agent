## 6. Interpretation

The repeated error is not that these rows lacked any evidence. They had event evidence. The error appears when a headline-valid Stage2 row is allowed to persist as Stage2-Actionable or progress toward Stage3 without a second bridge confirmation. Across the replay set, average 180D MFE was 21.7014%, but average 180D MAE was -45.6391%. That pattern means the signal often had optionality, but the entry/stage persistence rule was too forgiving.

```text
R13_STAGE2_FALSE_POSITIVE_REQUIRE_SECOND_BRIDGE_CONFIRMATION_OR_STAGE4B_OVERLAY_V125
```

Rule sketch:

```text
if trigger has only headline-valid Stage2 evidence
and delivery/revenue/margin/FCF/utilization/retention/cash bridge is missing
and MAE90 <= -20% or MAE180 <= -30%
then cap persistence at Stage2-Watch or add local Stage4B overlay
unless a second bridge confirmation appears before the MAE breach.
```
