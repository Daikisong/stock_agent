## 4. Research question

Loop87 deliberately tested many sector-specific “theme vs bridge” pairs:

```text
order backlog vs margin/cash bridge
test socket beta vs customer qualification/shipment bridge
AMPC/JV policy credit vs utilization and margin pass-through
strategic resource policy vs verified supply/customer/offtake bridge
retail rebound vs inventory/sell-through/gross-margin bridge
financial value-up extension vs incremental ROE/capital return bridge
bio data-event theme vs endpoint/partner/regulatory bridge
content/IP theme vs monetization/retention/margin bridge
mobility/auto-parts theme vs volume/mix/margin/cash bridge
construction policy theme vs PF/liquidity/backlog/cash bridge
nuclear policy theme vs project/legal/margin bridge
policy event headline vs direct beneficiary/asset/cash-flow bridge
```

The shared failure mode remains:

```text
theme label arrives first
business bridge is missing or proxy-only
MFE is small, unstable, or same-day price-only
MAE opens wide
```

R13 loop87 tests this diagnostic guard:

```text
Hard 4B guard:
bridge_missing_or_unverified
AND (
  (MFE90 < 10 AND MAE90 <= -20)
  OR
  (MFE90 < 20 AND MAE180 <= -25)
)
=> block Stage2-Actionable / Yellow / Green
=> route to Watch / 4B-risk / evidence-quality repair
```

A second price-only blowoff guard is added because loop87 includes several rows where same-day/event MFE is high but the missing bridge is exposed by MAE:

```text
Price-only event blowoff guard:
bridge_missing_or_unverified
AND MFE90 >= 20
AND MAE90 <= -20
=> do not count MFE as positive evidence
=> route to 4B-watch unless exact non-price bridge is repaired
```

Positive-control exception:

```text
If canonical bridge is present and MFE90 >= 20,
preserve Stage2/Yellow eligibility.
If MAE90<=-20 or MAE180<=-20 even with bridge present,
do not block Stage2 automatically, but keep Green strict and require exact evidence.
```
