## 4. Core finding

A valid public event is not the same as an accounting-trusted rerating path. Across contracts, EPC, PF repair, resource offtake, battery customer contracts, and JV/utilization routes, the profile should ask a second question before allowing Stage2/Stage3 persistence: **where is the cash, revenue recognition, margin, reserve quality, utilization, off-take clarity, or call-off clearance?**

```text
public evidence exists
  -> accounting/trust bridge still unconfirmed
  -> price path shows material MAE or weak MFE durability
  -> cap at Stage2-Watch or force local Stage4B
  -> if explicit trust/thesis break exists, route to Stage4C
```

This is a cross-archetype guardrail, not a new sector rule. It preserves true high-MFE rows as watch/local 4B but blocks clean Stage3 persistence when the bridge remains unverified.
