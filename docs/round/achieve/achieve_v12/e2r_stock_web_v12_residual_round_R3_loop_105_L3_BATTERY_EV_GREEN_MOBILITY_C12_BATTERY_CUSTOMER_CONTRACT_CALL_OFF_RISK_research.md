## 1. Selection Rationale

`V12_Research_No_Repeat_Index.md` still lists `C12_BATTERY_CUSTOMER_CONTRACT_CALL_OFF_RISK` as a low-coverage canonical archetype on the static ledger. Conversation-local work has already built C12 to the 50-row operating band, but the static index remains below the 50-row cleanup target because several recent rows are local handoff artifacts. This pass therefore adds a static-to-50 cleanup layer with canonical stage labels, emphasizing customer call-off, utilization, payment acceptance, inventory absorption, and cash conversion rather than generic battery vocabulary.

The residual failure mode is simple: the market often treats signed battery contracts, customer names, IRA/AMPC proximity, or equipment acceptance as if they were already demand. C12 should only score as durable when the order translates into released call-off, utilization, margin, and working-capital absorption. Otherwise the same narrative becomes a Stage2 cap, Local-4B watch, or 4C demand slowdown route.
