## 3. Research Thesis

C28 should not read like a dictionary of software/security keywords. A product can be security-related, cloud-related, API-related, or AI-related and still fail as a rerating engine if renewal, recurring billing, maintenance/service mix, customer lock-in, or operating-margin leverage is not visible. The useful gate is therefore:

```text
C28_RECURRING_CONTRACT_RETENTION_REQUIRES_RENEWAL_OR_BILLING_METRIC_AND_MARGIN_LEVERAGE_WITH_SCOPE_AND_MATURE_REVENUE_GUARDS
```

Mechanically, this is the same distinction as rent versus a one-time tool purchase. Contract retention is the rent stream; product identity is just the building sign. The model should pay rerating credit for the rent stream only when the invoices repeat and the margin bridge survives.
