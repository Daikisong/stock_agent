## 1. Scope / Why This C28 Follow-up Exists

C28 had only 28 representative rows in the static No-Repeat Index, and the earlier local C28 loop already separated ARR/renewal positives from product-theme proxies. This follow-up adds a different slice: cloud/IDC/security service renewals, captive ITO/vehicle software, B2B fintech SaaS, digital-ID/security platforms, network-security appliances, public-procurement security leadership, security-software product share, and data/AI SI proxies. The common residual is whether E2R should treat every software/security company as recurring-retention software. It should not. The gate is renewal or service recurrence plus margin leverage, not merely a product name.
