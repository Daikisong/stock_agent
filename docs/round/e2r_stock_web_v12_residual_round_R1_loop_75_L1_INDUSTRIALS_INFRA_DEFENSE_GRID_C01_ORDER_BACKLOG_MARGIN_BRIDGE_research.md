## 17. Sector-Specific Rule Candidate
Rule candidate: `R1_C01_shipbuilding_backlog_to_margin_conversion_gate`.

Proposal:
- Do not let backlog visibility alone promote a shipbuilding C01 case above Stage3-Yellow.
- Stage3-Green requires at least one of: confirmed margin bridge, confirmed revision, utilization/cost absorption signal, or repeat non-price customer/order quality.
- If backlog is high but margin_bridge_score < 4 and revision_score < 4, cap to Stage2-Watch or Stage2-Actionable, not Green.
- If MAE risk is already visible through sector timing and no margin bridge exists, apply `C01_high_MAE_orderbook_beta_watch_cap`.

This is sector-specific because shipbuilding orderbook duration is long; the orderbook is like a full dock, but earnings only launch when the vessel actually leaves the yard with margin attached.
