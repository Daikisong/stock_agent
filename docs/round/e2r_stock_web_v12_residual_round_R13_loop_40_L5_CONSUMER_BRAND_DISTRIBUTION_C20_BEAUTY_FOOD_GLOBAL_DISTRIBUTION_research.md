## 2. Round / Large Sector / Canonical Archetype Scope
This loop targets consumer-brand distribution after the prior L8 software-security residual loop. The specific C20 problem is that K-beauty winners behave like a three-stage conveyor belt: channel proof appears first, reorder/margin bridge follows, and only later do estimates fully catch up. A generic consumer-brand score can miss the first belt movement, while a naive theme score can mistake brand/channel noise for repeatable distribution leverage.

## 14. Stage2 / Yellow / Green Comparison
The core residual is not “Stage2 is earlier than Green,” which is already known. The new point is narrower: in C20, Stage2 should be promoted only when the evidence is distributor/ODM/reorder-like. Brand/channel-only cases should stay capped. Green lateness ratios were roughly 0.43 for 실리콘투, 0.77 for 브이티, and 0.71 for 코스메카코리아, showing that pure Green confirmation can miss 40-75% of the move from Stage2 to the first major peak.

## 17. Sector-Specific Rule Candidate
`sector_specific_rule_candidate = true`: For L5 consumer/brand distribution, do not equate “brand heat” with “distribution quality.” Stage2 promotion may receive a sector shadow bonus only when at least one of the following is present: repeat channel reorder, distributor operating leverage, ODM/customer mix margin bridge, or multi-region sell-through evidence.

## 18. Canonical-Archetype Rule Candidate
`canonical_archetype_rule_candidate = true`: For C20, add a shadow-only `c20_global_multi_channel_reorder_bridge` and `c20_distributor_operating_leverage_bonus`. Add a guard `c20_brand_channel_without_reorder_cap` that blocks Green when the evidence is only brand/channel narrative plus price strength.
