## 14. Stage2 / Yellow / Green Comparison

C32 does not behave like an earnings-revision archetype. Stage2 can appear early on tender or activism language; Yellow should require an executable bridge. Green should wait for control transfer plus a shareholder-value bridge. The Korea Zinc row is the exceptional positive case because the initial tender was not a cap; it became a live auction with escalating bids and float scarcity. SM, Hanssem, and Namyang show the opposite: once the event price, seller-only premium, or legal finality is known, the upside valve closes unless public holders receive a new route.

## 17. Sector-Specific Rule Candidate

```text
L10_CONTROL_EVENT_PRICE_CAP_AND_FAIRNESS_GUARD
```

- Treat policy/governance/control events as Stage2-watch until the execution route is visible.
- Upcredit only when public holders can monetize the control premium or when a control auction is demonstrably escalating.
- Downcredit seller-only control premiums, family-vote uncertainty, minority-unfriendly share swaps, and control restructurings where the public float becomes the payment currency.

## 18. Canonical-Archetype Rule Candidate

```text
C32_CONTROL_PREMIUM_REQUIRES_EXECUTABLE_TRANSFER_OR_ESCALATING_AUCTION_GATE
```

Rule mechanics:

```text
if event_type == public_tender_offer and competing_bid_or_float_lock == true: allow Stage2-Actionable to Stage3-Yellow credit
if event_type == public_tender_offer and no_competing_bid: treat tender_price as explicit 4B cap after local premium is priced
if event_type == seller_only_control_stake_sale: cap at Stage2-watch unless minority tender route or operating bridge exists
if event_type == family_vote_or_injunction_pending: cap at Stage2-watch and add legal-risk penalty
if event_type == minority_unfriendly_share_swap: route to Stage4B/Stage4C depending on fairness break severity
```
