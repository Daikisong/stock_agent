## 4. Interpretation

### 4.1 Positive paths

`003690` shows a clean C22 path: the reinsurance/rate-reserve bridge produced only shallow MAE through 180D while reaching +20.43% MFE. This is a better positive for C22 than a pure value-up label because it tests the capital-return/rate cycle through an insurer-specific reserve bridge.

`032830` is the strongest positive holdout. A late-2024 life-insurer trigger had +48.95% MFE180 and only -19.18% MAE180. The important point is not that rate-sensitive life insurers went up; it is that C22 should not over-penalize life-insurer rerating when the CSM/K-ICS/capital-return bridge is visible after the first value-up wave.

`000370` is useful because it is both a positive and a guardrail case. It reached +27.53% MFE90/180, but the subsequent -21.70% MAE180 and -38.60% post-peak drawdown mean C22 should keep a high-MAE/local-4B watch even for a working rate-cycle setup.

### 4.2 Counterexample / 4B / 4C paths

`000540` is the cleanest 4B watch. It had +51.03% MFE but also -23.80% MAE within 30D and a -49.55% post-peak drawdown. That is a local trading blowoff, not a full Yellow/Green insurance rerating without verified reserve quality.

`001450` is the hard-4C timing case. From the 2024-08-16 trigger it had only +6.85% MFE180 against -42.16% MAE180. This supports a narrower rule: C22 should route to 4C when reserve/loss-ratio thesis break is confirmed, but not merely because the generic insurance/rate label weakens.

`088350` is the life-insurer false positive. The rate-beta label produced only +2.53% MFE180 and -23.10% MAE180. It strengthens the requirement that life-insurer Stage2 needs CSM/K-ICS/capital-return confirmation, not just discount-rate sensitivity.

`005830` is a late-cycle drawdown guard rather than a pure failure. It eventually recovered into +19.30% MFE180 and +36.31% 1Y MFE, but first carried -28.77% MAE180. The current calibrated profile should avoid treating this as a clean Stage3 path without a drawdown/watch overlay.
